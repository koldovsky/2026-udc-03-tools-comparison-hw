===========================
Intelij Idea + Cursor
Ось конвенції цього проєкту згідно з app/AGENTS.md і фактичною структурою коду.
Стек
•TypeScript 5.6 з strict: true, target ES2022
•ESM-модулі ("type": "module" у package.json)
•Vitest 2.x для тестів (src/**/*.test.ts)
•Окремої збірки немає — лише утиліти в src/, без bundler/runtime
•Lint не налаштовано — ESLint/Prettier не підключені
Команди
Усі команди запускаються з каталогу app/:
Команда
Призначення
npm test
Тести (Vitest, одноразово)
npm run test:watch
Тести в watch-режимі
npm run typecheck
Перевірка типів (tsc --noEmit)
Перед завершенням змін: npm test має бути зеленим; за потреби — npm run typecheck.
Структура
app/
├── src/
│   ├── text-utils.ts       # slugify, truncate, parseTags
│   └── text-utils.test.ts  # unit-тести поруч із кодом
├── package.json
└── tsconfig.json
Нові утиліти — у text-utils.ts (або окремий файл у src/) за тим самим зразком: named export, тести в *.test.ts поруч.
Конвенції коду
1.Функціональний стиль — чисті функції з явними типами; без класів і глобального стану.
2.Стиль файлу — подвійні лапки, 2 пробіли відступу, camelCase для функцій і змінних.
3.Імпорти в тестах — ESM з суфіксом .js у шляху (from "./text-utils.js"), навіть якщо вихідний файл .ts.
4.Тести — один describe на функцію; назви it(...) описують поведінку; edge cases і регресії з тікетів — окремими тестами.
5.Публічний API — не змінювати сигнатури експортованих функцій (slugify, truncate, parseTags) без явної вимоги в тікеті; змінювати лише реалізацію й тести.
Guardrails
•Не ламати поведінку функцій, які вже покриті тестами (окрім навмисного фіксу з тікета).
•Не додавати залежності без потреби — проєкт мінімальний (лише typescript + vitest).
•Не комітити секрети, .env, ключі API.
•Коментарі — лише для неочевидної логіки.
•На Windows не використовувати 2>nul / >nul у shell-командах (створює файл nul).
Робота з задачами
•Баги/фічі приходять як тікети в materials/ (напр. task-bug-fix.md).
•Читати acceptance criteria; додавати регресійні тести на сценарії з тікета.
•Після правок коротко пояснювати, що змінено і чому.
На рівні всього homework-репо (кореневий AGENTS.md) додатково: артефакти класти в docs/ (tool-matrix.md, comparison.md тощо), документація українською або англійською, і перед PR — зелений cd app && npm test.
