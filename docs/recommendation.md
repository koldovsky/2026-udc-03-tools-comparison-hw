# Рекомендація для команди (Task D)

## Рекомендований основний (primary) інструмент

**Claude Code**

Чому: За результатами Task A та Task B Claude Code найкраще підходить як основний інструмент для щоденної розробки. Він працює як агент усередині IDE: самостійно знаходить потрібні файли, виконує багатоетапні задачі, додає регресійні тести та запускає перевірки. На практичному завданні баг був знайдений і виправлений з першого промпту, після чого інструмент сам перевірив результат запуском тестів.

## Рекомендований додатковий (secondary) інструмент

**ChatGPT** — для аналізу архітектури, підготовки документації, пояснення складної логіки, рев'ю рішень та генерації текстових артефактів.

Чому: ChatGPT добре працює як AI-партнер для обговорення рішень, аналізу альтернатив та підготовки документації. Він менш автоматизований у роботі з локальним кодом, але значно спрощує дослідження, навчання та підготовку супровідних матеріалів.

## Ключові компроміси (trade-offs)

* Вища автономність Claude Code потребує більшої довіри до автоматичних змін і регулярного перегляду запропонованого diff.
* ChatGPT забезпечує сильні пояснення та аналіз, але не має прямого доступу до локального workspace без додаткових інтеграцій.
* Єдиний `AGENTS.md` забезпечує хороший крос-інструментальний baseline, однак окремі інструменти можуть підтримувати додаткові власні можливості (MCP, hooks, custom commands тощо).
* Enterprise-тарифи покращують безпеку та керування даними, але збільшують загальну вартість використання.

## Ризик/застереження, за яким варто стежити

Основний ризик — **vendor lock-in**. Якщо команда почне активно використовувати специфічні можливості одного інструмента (наприклад, MCP, власні команди чи інші розширення), міграція на інший AI-інструмент може вимагати адаптації процесів та конфігурації.
