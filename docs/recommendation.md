# Рекомендація для команди (Task D)


## Рекомендований основний (primary) інструмент

**Claude Code CLI**

Виходячи зі швидкості виконання задачі, вартості токенів та якості результатів, для великих задач потрібно використоувати данний інструмент. Він дозволяє створювати агентів, розпаралелювати задачу, а також використовувати топові моделі за меньшу вартість.

## Рекомендований додатковий (secondary) інструмент

**GitHub Copilot** — для виконання швидких правок багів, з візуалізаціює в IDE, надає швидкий доступ до файлів, і надає доступ до моделей різних постачальників. Не використоувати для планування великих змін.

Чому: <2–3 речення>

## Ключові компроміси (trade-offs)

1. Автономність vs інтегрованість у редактор
- Claude Code CLI — агентний: сам читає файли, запускає команди, редагує кілька файлів і проганяє тести в одному ланцюжку («тікет → PR»). Сильний у multi-step задачах.
- GitHub Copilot — переважно inline-асистент (автодоповнення + чат у редакторі). Швидкий для дрібних правок «тут і зараз», але слабший в автономному виконанні складних багатокрокових задач (Copilot agent mode це частково закриває, але менш зрілий).

2. Ціна та ліміти
- GitHub Copilot — фіксована передбачувана підписка ($10/міс Individual, $19 Business), без явного розрахунку токенів. Дешевше й прогнозованіше.
- Claude Code — оплата за використання (токени) або через Max-підписку; на великих агентних сесіях вартість вища й менш передбачувана, зате видно розхід токенів.

3. Якість моделі та глибина міркувань
- GitHub Claude Code — свіжі моделі Claude (Opus/Sonnet), сильні у міркуванні над кодом і знаходженні справжньої причини бага, а не симптому.
- Copilot — вибір моделей (GPT, Claude, Gemini), але оптимізований під швидкість автодоповнення; глибина агентного міркування нижча.

4. Контекст кодової бази
- GitHub Copilot — глибока нативна індексація репозиторіїв у GitHub-екосистемі, знає PR/issues/історію.
- Claude Code — читає контекст «на льоту» через файлові інструменти + AGENTS.md; портативний, але без постійного індексу проєкту.

5. Екосистема / lock-in
- GitHub Copilot — щільно прив'язаний до GitHub/VS Code; enterprise-фічі (SSO, аудит, політики) зрілі.
- Claude Code — незалежний від IDE, працює в терміналі будь-де (CI, сервер); менший vendor lock-in, гнучкіше в pipeline.

6. Портативність контексту (AGENTS.md)
- Обидва читають AGENTS.md, але Claude Code покладається на нього як на основне джерело правил; Copilot історично тяжіє до .github/copilot-instructions.md, тож один файл не завжди «спільна мова» без дублювання.


## Ризик/застереження, за яким варто стежити

При виборі Claude Code CLI потрібно бути готовим до періодичної недоступності моделі, а також до швидких змін в правилах використання нових моделей.
