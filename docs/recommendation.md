# Рекомендація для команди (Task D)

> Синтез Task A (матриця) + Task B (одна задача BUG-101) + Task C (крос-tool `AGENTS.md`).

## Рекомендований основний (primary) інструмент

**GitHub Copilot** (у JetBrains IDE)

Чому: на нашій реальній задачі (BUG-101) Copilot з **першого промпту** знайшов
справжню причину, застосував лаконічний фікс + сфокусовані регрес-тести і **сам
одразу прогнав** `npm test`/`typecheck` (зелено) в один прохід — без мертвого коду.
Він так само підхопив `AGENTS.md` без налаштувань. За матрицею — зрілий agent-mode
із запуском команд, екосистема правил (`copilot-instructions.md`) і передбачувані
enterprise-гарантії (SSO/SOC 2, код не використовується для навчання на Business/
Enterprise) за прозорою ціною ($10/міс Pro).

## Рекомендований додатковий (secondary) інструмент

**Gemini** — для сценаріїв: великий контекст (до ~1M токенів) для широких
рефакторів і читання великих кодових баз, генерація ширшого набору тест-кейсів,
а також CLI-флоу (`gemini` CLI) для «тікет → PR» у скриптах/CI.

Чому: на BUG-101 Gemini теж дав коректний зелений фікс і навіть ширше тестове
покриття, але зайняв ~10 ітерацій, додав недосяжну гілку та **не зміг сам
запустити тести** в цьому IDE-інтерфейсі. Його сильні сторони — обсяг контексту й
щедрий безкоштовний tier, тому він добре доповнює primary у задачах «на об'єм».

## Ключові компроміси (trade-offs)

- **Ціна vs автономність:** Gemini має щедрий free tier і величезний контекст,
  але в цьому IDE-інтерфейсі обмежений набір інструментів (частина git-підкоманд,
  не запускає `npm test`); Copilot Pro ($10/міс) дає зрілий agent-mode із
  самостійним запуском команд і верифікацією.
- **Розмір контексту/індексація vs приватність/enterprise-tier:** Gemini —
  ~1M токенів контексту + GCP data residency/VPC-SC; Copilot — зрілий enterprise
  (SSO/SOC 2) і чітка політика «не тренуємось на вашому коді» для Business/Enterprise.
- **Крос-tool портативність vs глибина tool-specific фіч:** один `AGENTS.md`
  підхопили обидва інструменти (портативність підтверджена Task C), але глибші
  дії (agent сам запускає команди/тести) наразі сильніші в Copilot.

## Ризик/застереження, за яким варто стежити

**Залежність від можливостей агента в конкретному інтерфейсі.** Gemini у цьому
IDE не зміг сам запустити `npm test` — верифікацію довелося робити вручну. Тобто
«автономність» інструмента залежить не лише від моделі, а й від набору дозволених
дій в інтеграції. Варто стежити, які кроки агент реально автоматизує (запуск
тестів, git-операції), і за змінами лімітів/цінової політики free tier, перш ніж
робити інструмент обов'язковим у командному процесі.
