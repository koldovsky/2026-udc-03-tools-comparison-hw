# Рекомендація для команди (Task D)

Синтез Task A (матриця можливостей), Task B (той самий тікет у трьох
інструментах) і Task C (перевірка портативності AGENTS.md).

## Рекомендований основний (primary) інструмент

**Claude Code**

Чому: на тестовій задачі BUG-101 усі три інструменти дали ідентичний за
якістю фікс з першого промпту (Task B), тож вирішальними стали не «розумність»,
а робочий процес і екосистема. У проведеному bonus-запуску (Task E) Claude
Code закрив увесь цикл «тікет → фікс → тести → коміт → документи → PR» в
одному середовищі (для Codex і Cursor такий наскрізний сценарій ми не
тестували), має
найбагатшу систему розширень (skills, hooks, субагенти, MCP — Task A) і
бездоганно підхопив крос-tool `AGENTS.md` (Task C). Для команди, що працює
через термінал і git-процеси, це найменше тертя на щоденних задачах.

## Рекомендований додатковий (secondary) інструмент

**Cursor (Composer)** — для сценаріїв: робота розробників, які живуть в IDE;
рев'ю великих diff'ів у GUI; навігація по великій кодовій базі завдяки
семантичному індексу; онбординг новачків, яким CLI незвичний.

Чому: у Task B Cursor показав той самий результат (45 с, 1 промпт) із
найзручнішим візуальним рев'ю змін і прозорим звітом використання контексту.
Безкоштовний план Hobby дозволяє команді спробувати без бюджету, а
мультивендорний вибір моделей страхує від деградації якоїсь однієї.

Окрема примітка: **Codex CLI** — найдешевша точка входу (працює навіть із
ChatGPT Free) і розумний вибір для разових задач чи обмеженого бюджету, але як
командний стандарт поступається обом за екосистемою розширень.

## Ключові компроміси (trade-offs)

- **Ціна vs автономність:** Claude Code не має безкоштовного плану (від
  $20/міс за місце), тоді як Codex/Cursor можна почати за $0 — але саме
  Claude Code дає найповнішу агентну автономність end-to-end.
- **Індексація vs приватність:** семантичний індекс Cursor прискорює роботу
  з великим репо, але код завантажується на сервери Cursor; CLI-агенти
  досліджують код на льоту і нічого постійно не зберігають.
- **Портативність AGENTS.md vs tool-specific глибина:** Task C показав, що
  один узагальнений `AGENTS.md` підхоплюють усі три інструменти — тому варто
  тримати правила саме там, а tool-specific файли (`.cursor/rules`,
  `CLAUDE.md`) використовувати лише як тонкі надбудови. Це зменшує vendor
  lock-in незалежно від вибору primary.

## Ризик/застереження, за яким варто стежити

**Зміна цінової політики та лімітів.** Цінові політики та ліміти всіх трьох
вендорів можуть змінюватися (вікна 5 годин, місячні квоти, вартість
seat-планів). Для команди це прямий бюджетний ризик: варто раз
на квартал переглядати фактичне споживання (у Codex — `/status`, у Cursor —
dashboard Usage, у Claude Code — звіти підписки) і тримати `AGENTS.md`
tool-agnostic, щоб міграція на інший інструмент коштувала години, а не тижні.
