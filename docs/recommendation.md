# Рекомендація для команди (Task D)

> Синтез Task A + B + C.

## Рекомендований основний (primary) інструмент

**Cursor** (agentic IDE, Pro $20/міс)

Чому: для щоденної інтерактивної розробки Cursor дає найповніший шар — постійний
embeddings-індекс кодової бази, `@`-згадки, Tab-автодоповнення та Agent mode в
звичному IDE-контексті (матриця, Task A). На задачі BUG-101 результат такий самий
як у Claude Code (1 промпт, коректний фікс, 2 регресійні тести, `npm test`
зелений), але Cursor зручніший для команди, що вже живе в редакторі: diff
видно одразу, тести запускаються з того ж вікна, мінімальний поріг входу для
розробників без звички до CLI-агентів. Мультивендорний вибір моделей
(Claude/GPT/Gemini + Composer/Auto) знижує залежність від одного постачальника
моделі. Узагальнений `app/AGENTS.md` підхоплюється автоматично як workspace rule
(Task C) — достатньо одного крос-tool baseline для щоденних конвенцій.

## Рекомендований додатковий (secondary) інструмент

**Claude Code** (CLI-агент, Pro $20/міс) — для сценаріїв:

- **CI/CD і автоматизація** — headless-режим `claude -p`, hooks, subagents;
  агент легко вбудовується в скрипти, pre-commit, GitHub Actions без IDE.
- **Великі автономні задачі** — коли потрібен довгий агентний цикл (план →
  правки → тести → git) без постійної присутності в редакторі.
- **Швидкі правки з терміналу** — на BUG-101 Claude Code впорався за ~2 хв
  проти ~5 хв у Cursor при ідентичній якості фіксу; на добре сформульованих
  тікетах CLI-агент ефективний як «одна команда — готовий PR».
- **Робота без IDE** — SSH-сесії, віддалені сервери, pairing з тим, хто не
  використовує Cursor.

Чому: Claude Code не конкурує з Cursor за «щоденний драйвер», а доповнює його
там, де IDE-шар зайвий. Глибша агентна обв'язка (skills, hooks, MCP-сервер)
корисна для кастомізації пайплайнів команди. На Task B агент сам відстежив
git-статус і нагадав про методологію порівняння — краще тримає контекст репо
поза межами конкретного тікета.

## Ключові компроміси (trade-offs)

- **Ціна vs автономність:** стартові плани однакові ($20/міс), але верхні tier'и
  рахують по-різному — у Cursor usage-кредити на сторонні моделі (оновлення
  червня 2026), у Claude Code — множники лімітів однієї екосистеми Anthropic.
  Для команди з 5–10 людей варто рахувати не seat, а сумарний monthly usage.
- **Якість індексації vs приватність:** Cursor синхронізує embeddings-індекс на
  сервер (US) — швидший семантичний пошук, але код покидає локальну машину
  (Privacy Mode / Teams зменшують ризик, але індекс лишається серверним).
  Claude Code шукає контекст на вимогу (grep/glob) — менше «магії», більше
  контролю над тим, що відправляється в модель.
- **Крос-tool портативність (`AGENTS.md`) vs глибина tool-specific фіч:** один
  узагальнений `app/AGENTS.md` працює в обох інструментах без дублювання в
  `.cursor/rules` чи `CLAUDE.md` (Task C). Нюанс: шари контексту різні —
  Cursor бачить і кореневий `AGENTS.md` репо, Claude Code із робочої
  директорії `app/` лише nested-файл, тому baseline треба писати
  самодостатнім. Крім того, skills, hooks, slash-команди, MCP-конфіг —
  лишаються per-tool; спільний baseline не замінює tool-specific
  налаштування для просунутих сценаріїв.
- **Мультивендорність vs екосистемна глибина:** Cursor дозволяє порівнювати
  Claude/GPT/Gemini в одному вікні; Claude Code прив'язаний до Anthropic, зате
  дає найглибшу інтеграцію з їхніми моделями (permission modes, subagents,
  Compliance API на Enterprise).
- **Якість задачі vs вибір інструмента:** на BUG-101 обидва інструменти дали
  ідентичний результат за 1 ітерацію — на добре сформульованому тікеті з
  acceptance criteria постановка задачі важить більше, ніж IDE чи CLI.

## Ризик/застереження, за яким варто стежити

**Зміна цінової політики Cursor і непрозорість usage.** У червні 2026 Cursor
перейшов на модель usage-кредитів з окремими пулами для власних і сторонніх
моделей (Teams: Standard $40 vs Premium $120 seat). Для команди це означає
ризик непередбачуваних витрат: розробник, що часто перемикається на преміум-
моделі або Background Agents, може вичерпати пул швидше за очікуване.
Рекомендуємо щомісячно знімати usage-звіт (Teams admin analytics) і зафіксувати
team policy: коли Auto/Composer, коли преміум-модель, коли Claude Code через
API замість Cursor usage-кредитів.
