# Рекомендація для команди (Task D)

Це підсумок трьох попередніх кроків: таблиці можливостей
([`tool-matrix.md`](./tool-matrix.md), Task A), реального тесту на одній
задачі ([`comparison.md`](./comparison.md), Task B) і перевірки спільного
файлу-інструкції ([`agents-md-notes.md`](./agents-md-notes.md), Task C).

## Коротко для тих, хто не хоче читати все

Уявіть двох помічників у команді:

- **Cursor** — досвідчений редактор, що сидить поруч за столом. Бачить
  увесь проєкт одразу, підказує на льоту, дає обрати "чиїм мозком" думати.
  Ідеальний для щоденної роботи руками в коді.
- **Claude Code** — надійний підрядник, якому даєш завдання й він приносить
  готовий результат сам, без нагляду. Ідеальний, коли треба щось
  автоматизувати або зробити "під ключ".

**Наш вибір:** Cursor — основний інструмент (щоденна робота), Claude Code —
додатковий (автоматизація та особливі випадки). Нижче — чому саме так,
простими словами.

---

## Основний (primary) інструмент: **Cursor**

**Чому основний?** Тому що 90% часу команда робить те, що Cursor вміє
найкраще: сидить у редакторі коду й вносить зміни руками, з AI-підказками
під рукою.

Три причини мовою фактів:

1. **Він "знає проєкт напам'ять".** Cursor заздалегідь сканує весь код і
   будує внутрішню карту (як зміст книги). Коли ставиш питання — знаходить
   потрібне місце миттєво, бо вже все "прочитав". Для великого проєкту це
   економить купу часу щодня.
2. **Дає обрати "мозок" під задачу.** Можна перемикати AI-модель (Claude,
   GPT, Gemini) або довірити вибір інструменту. Гнучкість, якої немає в
   Claude Code.
3. **На реальному тесті не програв.** У Task B ми дали обом інструментам
   ту саму задачу (виправити баг у функції `truncate`). Результат —
   **нічия**: обидва знайшли справжню причину з першої спроби, обидва
   виправили однаково, жодних поломок. Висновок важливий: якщо на однаковій
   задачі якість однакова, то основним варто робити той, що **зручніший
   щодня** — а це Cursor.

## Додатковий (secondary) інструмент: **Claude Code**

**Для яких випадків:** коли завдання треба доручити повністю й отримати
готовий результат без відкритого редактора — наприклад, "візьми оцей тікет
і зроби готовий Pull Request" (це окремо перевіряється в Task E). Також:
автоматизація в конвеєрі складання (CI/CD), задачі з підвищеною
приватністю коду, побудова власних повторюваних AI-процесів.

**Чому саме він у цій ролі:** у таблиці можливостей (Task A) Claude Code
має найбагатший "конструктор" для автоматизації — підключувані навички,
кілька помічників з різними ролями одночасно, автоматичні реакції на події.
Cursor цього не має. А Task C підтвердив приємну річ: спільний файл-
інструкція `AGENTS.md` однаково добре працює в обох інструментах — тобто
перемикатися між ними **безпечно**, правила проєкту не губляться.

## Ключові компроміси (за що платимо, коли вибираємо)

Кожен вибір має зворотний бік. Ось головні "гойдалки":

- **Дешевше на старті ⇄ передбачуваність рахунку.**
  Cursor стартує дешевше (≈ $20/міс), але при активному використанні
  докидає плату "за фактичне споживання". Claude Code у розширеному плані
  дорожчий ($100–200/міс), зате дає великий ліміт наперед — менше сюрпризів
  у рахунку під час довгих автоматичних сесій.

- **Швидкий пошук по коду ⇄ приватність.**
  Готова "карта проєкту" Cursor зручна, але щоб її побудувати, код за
  замовчуванням вирушає на сервери Cursor (є режим приватності, щоб цього
  уникнути). Claude Code нічого не сканує наперед — менше даних покидає
  комп'ютер, але ціна цього — повільніший і дорожчий пошук на дуже великих
  проєктах.

- **Свобода переключатись ⇄ глибина унікальних фіч.**
  Спільний `AGENTS.md` (Task C) дозволяє вільно ходити між інструментами
  без втрати правил проєкту — це плюс. Але просунуті можливості Claude Code
  (навички, помічники, автоматизації) в цьому файлі не виражаються і в
  Cursor не переносяться. Тобто якщо команда збудує процеси на цих фішках —
  вона прив'язується до Claude Code саме для них.

- **Вибір моделі ⇄ стабільність.**
  Cursor дає обрати модель під настрій — гнучко, але якість "плаває"
  залежно від вибору. Claude Code працює з однією родиною моделей —
  результат стабільніший і легше передбачуваний, що важливо для
  автоматизації, де потрібна відтворюваність.

## Один ризик, за яким варто стежити з першого дня

**Рахунок може непомітно "втекти" вгору.**

Обидва інструменти при активному використанні виходять за межі фіксованої
підписки й починають рахувати "за фактом" (за токени — одиниці, якими
міряється обсяг тексту, що обробляє AI). Небезпека в тому, що це
відбувається **тихо**: команда працює як звикла, а наприкінці місяця
приходить рахунок, більший за очікуваний — особливо якщо активно
використовувати автоматичні сесії Claude Code.

**Що робити (конкретно):** не чекати місячного інвойсу, а з першого тижня
дивитись на витрати щотижня per людину:
- у Cursor — розділ **Settings → Usage** (або дашборд на cursor.com),
- у Claude Code — команда `/cost` у сесії + дашборд у Console.

І виставити собі поріг-сигнал ("якщо за тиждень більше $X — розібратись"),
а не покладатись на те, що "якось воно буде".
