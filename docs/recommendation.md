Чому: Згідно Task A, Claude Code забезпечує найвищу локальну агентну автономність з контекстним вікном до 1 млн токенів та ефективним кешуванням підказок. Task B підтвердив, що Haiku 4.5 виконав баг-фікс найшвидше (2м 22сек) за мінімальну вартість ($0.0698), успішно знайшовши справжній баг, виправивши код та додавши регресійні тести за одну ітерацію без регресій. ## Рекомендований додатковий (secondary) інструмент

**GitHub Copilot CLI** — для сценаріїв: швидкі локальні правки, робота з GitHub Issues/PR, мінімізація вартості ($0.05 за операцію), тестування різних моделей (OpenAI vs Anthropic).

Чому: Task A демонструє, що Copilot CLI пропонує унікальну гнучкість вибору моделей та безшовну інтеграцію з GitHub. Task B показав, що він виконав цю саму задачу за 2м 30сек з найнижчою вартістю. Ideal для команд, які вже повністю вбудовані в GitHub-екосистему або мають жорсткі бюджетні обмеження.

## Ключові компроміси (trade-offs)

- **Ціна vs. Глибина автономності**: Claude Code Opus 4.8 коштує у 7–10 разів дорожче ($0.49), але дає вищу якість на складних архітектурних завданнях. Компроміс: Haiku для стандартних задач, Opus лише для справді складних сценаріїв.

- **Приватність vs. Гнучкість моделей**: GitHub Copilot (Free/Pro) за замовчуванням передає дані для навчання; Claude має строгу політику конфіденційності навіть на персональних планах. Компроміс: для конфіденційних проектів вибирати Claude; для відкритого коду допускати Copilot.

- **Локальна швидкість vs. Хмарна індексація**: Claude Code залежить від локального grep; Copilot використовує централізовану хмарну індексацію GitHub. Компроміс: Claude для чистого локального розроблення; Copilot для enterprise-репозиторіїв.

- **Агентна культура**: Claude Code запитує дозвіл на комміти; Copilot діє на автопілоті. Компроміс: обирати за культурою процесу команди (рецензія vs. швидкість).

## Ризик/застереження, за яким варто стежити

**Vendor lock-in на Claude Anthropic**: Якщо команда повністю рухнеться на Claude Code, у разі змін цінової політики або припинення обслуговування сервісу Anthropic переведення на альтернативний інструмент буде затратним. Мітигація: дотримуватися крос-інструментного AGENTS.md, щоб Copilot CLI залишався резервним варіантом. Також слідкувати за змінами цін Anthropic, особливо для Opus 4.8, який уже дорогий.

## Рекомендований додатковий (secondary) інструмент

**Claude Code** — для сценаріїв: великі локальні рефактори, автономні agent-цикли, складні аналізи з великим контекстним вікном.

Чому: Claude Code має значну перевагу в автономності та великому контекстному вікні, що робить його ефективним для довготривалих локальних сесій (рефакторинг, міграції, автономні виправлення), як свідчать результати в матриці та порівнянні.

## Ключові компроміси (trade-offs)

- Ціна vs автономність: Copilot дешевший і краще інтегрований з GitHub; Claude дорожчий, але потужніший для автономних задач.
- Приватність і enterprise-функції: для відповідності compliance варто використовувати корпоративні плани і налаштування, які гарантують, що дані організації не використовуються для навчання моделей.
- Портативність vs глибина автоматизації: використання інструмент-специфічних agent-фіч покращує ефективність, але зменшує переносимість робочих процесів між інструментами.

## Ризики та застереження

- Vendor lock-in та зміни цінової політики: мати план на випадок зміни тарифів або функціональних обмежень.
- Залежність від мережі/хмари: забезпечити локальні процедури для критичних операцій.
- Compliance/audit: перевіряти налаштування приватності й використовувати корпоративні підписки для чутливих даних.
