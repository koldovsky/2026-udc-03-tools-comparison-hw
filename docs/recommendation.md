# Рекомендація для команди (Task D)

Синтез Task A (матриця) + Task B (задача на fix бага) + Task C (AGENTS.md).

## Рекомендований основний (primary) інструмент

**Claude Code**

Чому: за матрицею (Task A) Claude Code — єдиний з двох інструментів, де
агентна автономність термінально-нативна «з коробки»: повний доступ до
shell/git/тестів, subagents, checkpoint/resume для довгих задач і CI-
інтеграція без додаткового плану чи «слайдера автономності», як у Cursor.
Весь репозиторій одразу доступний у 1M-токенному контексті без окремого
embedding-кроку, а privacy-політика прозоріша за замовчуванням (no-training,
ZDR-опція, SOC 2 Type I&II, ISO 42001, HIPAA BAA — без потреби переходити на
вищий tier). На практичній задачі (Task B) Claude Code також дав чистіший
результат: один прохід Read → Edit → `npm test` без повторних запусків і без
sandbox-friction, який зустрівся в Cursor (EPERM на `node_modules` при
першому запуску тестів). Обидва інструменти знайшли ту саму справжню
причину бага й дали ідентичний за суттю фікс без регресій — тобто якість
рішення однакова, а різниця саме в надійності автономного виконання, що і є
ключовим критерієм для «основного» інструменту команди.

## Рекомендований додатковий (secondary) інструмент

**Cursor Desktop** — для сценаріїв: повсякденний inline pair-programming з
редактором, великі рефактори з візуальним review диффів по кроках, швидкий
вибір моделі під задачу (GPT-5.x / Gemini / Claude в одному вікні), і будь-
яка робота, де корисний семантичний пошук по кодовій базі (`@codebase`).

Чому: у Task A Cursor виграє за мульти-провайдерним вибором моделей і
вбудованим у редактор inline-review, що краще підходить для ітеративної
розробки з людиною в циклі, ніж повністю автономний CLI-агент. У Task C
Cursor показав додаткову цінність саме для «швидких» задач: підхопивши той
самий `AGENTS.md`, він самостійно підтягнув конкретні code-цитати
(`slugify`, тестові блоки) як ілюстрацію конвенцій — тобто краще працює як
контекстний асистент «на льоту» в IDE, ніж Claude Code, який обмежився
переказом файлу.

## Ключові компроміси (trade-offs)

- **Ціна vs автономність** — на однакових $20/міс (Pro) Claude Code дає
  глибшу наскрізну автономність (термінал, subagents, CI), а Cursor —
  ширший вибір моделей і безкоштовний Hobby-тір, але агентність обмежена
  «слайдером автономності» та інтерфейсом редактора.
- **Якість індексації vs приватність** — Cursor будує embedding-індекс
  репозиторію на сервері (plaintext видаляється, але ембединги й метадані
  зберігаються) заради ефективнішого семантичного пошуку; Claude Code
  тримає весь код у контексті моделі без окремого індексу й пропонує ZDR,
  що простіше для команд із суворими вимогами до даних.
- **Портативність AGENTS.md vs глибина tool-specific фіч** — Task C
  показав, що один узагальнений `AGENTS.md` однаково коректно підхоплюється
  обома інструментами без налаштувань, тож команда не прив'язана до
  формату одного вендора. Але Cursor поверх цього додає власну цінність
  (semantic `@codebase`, inline code-цитати), яку неможливо відтворити
  лише файлом правил.
- **Enterprise-готовність vs гнучкість моделей** — Claude Code вже на
  базовому рівні має SOC 2 I&II, ISO 42001, HIPAA BAA; у Cursor подібний
  рівень (аудит-логи, DPA, контроль моделей/репозиторіїв) відкривається
  лише на Business/Enterprise-тарифі.

## Ризик/застереження, за яким варто стежити

**Vendor/model lock-in у Claude Code**: на відміну від Cursor, який
абстрагує вибір моделі (Auto-режим між провайдерами), Claude Code підтримує
лише моделі Anthropic. Якщо як команда ми зробимо Claude Code основним
робочим інструментом, будь-яка зміна цінової політики Anthropic, ліміт
доступу (rate limits за $20/$100/$200 тарифами) або тимчасова відмова
сервісу безпосередньо зупиняє роботу команди без вбудованого fallback на
іншого провайдера — це варто закривати організаційно (напр. тримати Cursor
або прямий API-доступ як запасний канал), а не лише технічно.
