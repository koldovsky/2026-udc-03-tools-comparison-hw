# Рекомендація для команди (Task D)

## Рекомендований основний (primary) інструмент

**Claude Code**

З матриці (Task A) видно, що Claude Code єдиний із двох інструментів, хто підтримує повний агентний цикл: читає файли, пише код, запускає тести і відкриває PR без ручних підтверджень. На практичній задачі (Task B) це підтвердилось — від вставки тікета до зелених тестів пройшло ~1 хвилина й один промпт, тоді як Cursor потребував 2 ітерації та ручних «Accept». З Task C з'ясувалось, що Claude Code автоматично підхоплює `AGENTS.md` і дотримується всіх конвенцій без додаткових нагадувань — це важливо для стабільності стилю коду в команді. За рівної ціни ($20/міс) і ширших можливостей автоматизації через MCP і hooks — вибір на його користь очевидний для задач, де важлива автономія.

## Рекомендований додатковий (secondary) інструмент

**Cursor** — для сценаріїв: швидкі правки прямо в IDE без виходу в термінал, code review з inline-коментарями, навігація по великій кодовій базі через `@codebase`

Cursor зручніший там, де розробник хоче бачити кожну зміну перед застосуванням — це виправдано при роботі з наприклад новим кодом. Індексація репозиторію дає помітну перевагу у великих репозиторіях, де Claude Code без явного контексту змушений проходитись по файлам вручну.

## Ключові компроміси (trade-offs)

- **Автономність vs контроль:** Claude Code виконує задачі без підтверджень — швидко. Cursor зупиняється після кожного кроку — повільніше, але передбачуваніше для критичних задач.
- **Ціна vs передбачуваність витрат:** Claude Code через API тарифікується за токенами — при інтенсивному використанні витрати важко передбачити. Cursor Pro — фіксований $20/міс незалежно від обсягу.
- **Індексація кодової бази vs портативність:** Cursor має вбудованиу індексацію коду і добре орієнтується у великих репозиторіях; Claude Code читає файли на запит — гнучкіше, але повільніше без явного контексту.
- **Крос-tool портативність AGENTS.md vs глибина tool-specific фіч:** Один `AGENTS.md` покриває Claude Code повністю, але для Cursor потрібне дублювання в `.cursor/rules`.

## Ризик/застереження, за яким варто стежити

**Залежність від цінової політики Anthropic.** Claude Code на Pro-плані має ліміти, які Anthropic може змінити в будь-який момент. Якщо команда глибоко інтегрує Claude Code в CI/CD-процеси через API, зростання токен-витрат або зміна тарифів може несподівано збільшити щомісячні витрати. Варто з самого початку встановити моніторинг токен-використання і мати резервний план (Cursor або локальна модель) для  автоматизованих задач.
