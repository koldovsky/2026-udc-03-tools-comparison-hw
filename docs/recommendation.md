# Рекомендація для команди (Task D)

Синтез Task A (`docs/tool-matrix.md`) + Task B (`docs/comparison.md`) + Task C
(`docs/agents-md-notes.md`).

## Рекомендований основний (primary) інструмент

**Cursor**

Чому: на BUG-101 (Task B) обидва інструменти знайшли справжню причину бага
й закрили задачу з першого проходу з ідентичною якістю фіксу (8/8 тестів),
але Cursor зробив це за 1 промпт і ~3 хв проти 2 промптів і ~5 хв у Claude
Code — для щоденної роботи команди (тікет у чаті → diff одразу в редакторі →
тести у вбудованому терміналі) це відчутна різниця в кількості контекст-
перемикань. Матриця (Task A) додає до цього мульти-модельний вибір «на
льоту» (GPT-5.x/Claude/Gemini/Grok в одному Agent mode) і готову семантичну
індексацію (`@codebase`) — на крихітному `app/` перевага не проявилась, але
на великій малознайомій кодовій базі команди вона стане відчутною. Cursor
також автоматично підхопив `AGENTS.md` як постійне workspace rule без
додаткових налаштувань (Task C), що знижує поріг входу для нових учасників
команди.

## Рекомендований додатковий (secondary) інструмент

**Claude Code** — для сценаріїв: headless-автоматизація «тікет → PR» у CI,
великі рефактори через subagents/TodoWrite-пайплайни, швидкі точкові правки
з терміналу без відкриття IDE.

Чому: на Task B Claude Code пройшов верифікацію тестами з першого запуску
`npm test` без відкатів (на відміну від Cursor, де sandbox зажадав повторного
запуску через EPERM), і його CLI-нативний loop (`find`/`Edit`, TodoWrite,
subagents, hooks) явно розрахований на скриптовані/CI-сценарії, а не на
інтерактивну роботу в редакторі. Enterprise-приватність теж контрольованіша
за замовчуванням (API/Enterprise-дані не йдуть у тренування, data residency
через Bedrock/Vertex) — це важливо саме для CI-раннерів і автоматизованих
пайплайнів, де немає людини, яка вручну вмикає Privacy Mode.

## Ключові компроміси (trade-offs)

- **Ціна vs автономність** — Cursor Pro (~$20/міс) дає мульти-моделі, але без
  прозорого лічильника токенів/вартості за запит; Claude Code Pro/Max
  ($20–200/міс) або pay-as-you-go по API дає точніший контроль над
  автономними/довгими агентними прогонами (subagents, hooks), але за
  окрему плату понад базову підписку.
- **Якість індексації vs приватність/enterprise-tier** — семантична
  індексація Cursor (`@codebase`) пришвидшує орієнтацію на великих
  репозиторіях, але означає додатковий шар обробки коду на боці Cursor;
  Claude Code без embeddings-індексу шукає «на льоту» (ripgrep/glob), що
  повільніше на великих кодових базах, зате простіше узгодити з
  enterprise-вимогами до resid­ency/приватності (Bedrock/Vertex, zero data
  retention).
- **Портативність `AGENTS.md` vs глибина tool-specific фіч** — обидва
  інструменти коректно підхоплюють спільний `AGENTS.md` без дублювання
  (Task C), тому крос-tool baseline виправданий і дешевий у підтримці; але
  tool-specific шари (`.cursor/rules`, `CLAUDE.md`, Skills/MCP-конфіги)
  дають глибші можливості (напр. subagents і hooks у Claude Code, вбудований
  diff-review у Cursor), які не переносяться між інструментами й вимагають
  окремої підтримки.
- **UX/середовище vs headless-надійність** — Cursor зручніший для
  щоденної інтерактивної роботи (візуальний diff, вбудований термінал), але
  його sandbox іноді вимагає ручного/автоматичного retry команд; Claude Code
  пройшов верифікацію з першого разу, що краще підходить для некерованих
  CI-прогонів, де немає кому підтвердити retry.

## Ризик/застереження, за яким варто стежити

**Vendor lock-in через tool-specific шари поверх спільного `AGENTS.md`.**
Наразі команда покладається на `AGENTS.md` як портативний baseline, але
частина цінності кожного інструменту (Cursor: `.cursor/rules`, MCP-конфіги,
Skills у `.cursor/skills/`; Claude Code: subagents, hooks, slash-commands у
`.claude/`) живе в конфігах, які не переносяться між інструментами. Якщо
команда з часом наростить багато логіки саме в цих tool-specific шарах (а не
в `AGENTS.md`), відмова від інструменту або зміна його цінової
політики/лімітів стане дорожчою, ніж зараз. Варто регулярно перевіряти, що
критичні guardrails і конвенції залишаються в `AGENTS.md` (а не тільки в
`.cursor/*` чи `.claude/*`), і не покладатись на функції одного вендора там,
де є прийнятна крос-tool альтернатива.
