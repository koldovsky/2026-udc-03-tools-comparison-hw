# Рекомендація для команди (Task D)

Синтез Task A (матриця) + Task B (порівняння на задачі) + Task C (крос-tool AGENTS.md).

---

## Рекомендований основний (primary) інструмент

**Claude Haiku 4.5** (Claude Code IDE)

Чому: на реальній задачі bug-fix (Task B) Haiku знайшов справжню кореневу причину бага й надав робочий фікс з регресійними тестами — при вартості $0.20 (20 центів) проти $0.50 (50 центів) у Sonnet (в 2.5 рази дешевше). За матрицею (Task A) обидва інструменти ідентичні за платформою, автономністю, індексацією та підтримкою AGENTS.md — різниця лише в ціні й глибині reasoning. На основі однієї bug-fix задачі (Task B) Haiku забезпечує достатній рівень якості для типових bug-fix та простих фіч без переплати за потужність. **Застереження:** висновок про достатність Haiku для щоденних завдань заснований на одній bug-fix задачі — для підтвердження цього правила потрібні результати на ширшому наборі різноманітних завдань (рефактори, архітектурні рішення, code review складних модулів).

---

## Рекомендований додатковий (secondary) інструмент

**Claude Sonnet 4.6** (Claude Code IDE) — для сценаріїв: складні рефактори з неочевидними залежностями, архітектурні рішення з широким контекстом, задачі де потрібна надійна перша спроба (ревʼю перед мержем, PR у продакшн), дослідження незнайомої кодової бази.

Чому: на Task B Sonnet завершив за 1 чисту ітерацію без помилок — явно озвучив причину бага перед кодом, написав компактнішу реалізацію та акуратніші тести. На Task C додав 4 тести замість 3 і використав явніший API (`charAt` vs `text[0]`). Ця педантичність виправдана, коли ціна помилки або повторної ітерації вища за різницю у вартості токенів ($0.30 за сесію).

---

## Ключові компроміси (trade-offs)

- **Вартість vs надійність першої спроби:** Haiku дешевший у 2.5×, але інколи потребує другої ітерації (Task B: не врахував edge case з першого разу). Sonnet надійніший за 1 хід, але кожна сесія коштує більше.
- **Однорідність платформи vs диверсифікація:** обидва на одній платформі Claude Code — зручно (один білінг, один AGENTS.md), але без диверсифікації постачальника.
- **Крос-tool портативність AGENTS.md vs tool-specific глибина:** один `app/AGENTS.md` однаково підхоплюється обома — перевага над IDE-специфічними rules (Cursor, JetBrains), де треба підтримувати окремі файли.
- **Однорідність моделей vs спеціалізація:** немає сенсу завжди використовувати Sonnet — для добре описаних задач якість однакова. Потрібна явна командна угода: «Haiku для рутини, Sonnet для складного».

---

## Ризик/застереження, за яким варто стежити

**Vendor lock-in + одночасна зміна цінової політики Anthropic.** Оскільки обидва інструменти (primary та secondary) — це різні моделі однієї платформи, будь-яка зміна умов (підвищення цін, скасування плану, зміна доступних моделей) одночасно вдарить по всьому AI-тулінгу команди. Anthropic вже змінював моделі між версіями (Haiku 3 → 4.5, Sonnet 3.5 → 4.6). Рекомендується раз на квартал перевіряти актуальне ціноутворення та мати резервний варіант (Aider + OpenAI або Gemini Code Assist) для критичних сценаріїв.
