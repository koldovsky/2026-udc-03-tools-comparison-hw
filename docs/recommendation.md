# Рекомендація для команди (Task D)

> Синтез Task A (матриця) + Task B (задача на баг) + Task C (AGENTS.md).

## Рекомендований основний (primary) інструмент

**Claude Code (Opus 4.8)**

Чому: за матрицею (Task A) він поєднує найсильнішу дефолтну модель (Opus 4.8 на
`xhigh`, 1M контекст) зі зрілими примітивами автономності — субагенти,
детерміновані Workflows, hooks, фонові задачі — і працює переносимо в терміналі
(CI, віддалений сервер, будь-яка IDE), а не лише в одній IDE. На задачі (Task B)
він з першого промпту (тікет «як є») знайшов **справжню причину** бага
`truncate()` (суфікс не враховувався в межі `maxLength`), обробив edge case і
додав регресійні тести без регресій — тобто на якість фіксу можна покластися.
Enterprise-профіль (SOC 2 Type II, SSO/SCIM, data residency через Bedrock/Vertex)
робить його прийнятним для командного впровадження.

## Рекомендований додатковий (secondary) інструмент

**Junie (JetBrains)** — для сценаріїв: глибока IDE-робота в IntelliJ/WebStorm
(рефактори з опорою на символьні індекси), команди/розробники, яким потрібна
гнучкість провайдерів моделей (BYOK: Anthropic/OpenAI/Google/xAI), і швидкі
правки без виходу з JetBrains-екосистеми.

Чому: на задачі (Task B) Junie дав **той самий якісний фікс**, що і Claude Code,
а його IDE-індекси (Task A) дають точний контекст «безкоштовно» всередині IDE.
Для JetBrains-центричних воркфлоу він швидший і дешевший ($8–16 у JetBrains AI),
тож логічно тримати його як другий інструмент для роботи всередині IDE.

## Ключові компроміси (trade-offs)

- **Ціна vs автономність:** Claude Code дорожчий (Opus 4.8 $5/$25 за 1M токенів,
  або Max $100–200/mo), але дає найсильнішу модель + автономні примітиви; Junie
  дешевший і гнучкіший за моделями, але з простішою агентністю.
- **Якість індексації vs портативність:** IDE-індекси Junie = точний контекст,
  але прив'язка до JetBrains; агентний пошук Claude Code (Grep/Glob/Read) =
  портативний (CI, будь-яка IDE), ціною реконструкції контексту щоразу (частково
  компенсується prompt caching).
- **Крос-tool AGENTS.md vs глибина tool-specific фіч:** один `AGENTS.md`
  підхопили обидва інструменти без tool-specific файлів (Task C), але глибші
  можливості (hooks/Workflows/субагенти в Claude Code; IDE-рефактори в Junie)
  лишаються прив'язаними до конкретного інструмента.
- **Приватність/enterprise-tier:** обидва не тренуються на даних за замовчуванням,
  ZDR доступний для enterprise; проте окремі моделі (напр. Fable 5) вимагають
  ≥30-денного retention і несумісні з ZDR.

## Ризик/застереження, за яким варто стежити

**Vendor/модельний lock-in на Anthropic у Claude Code.** На відміну від Junie,
Claude Code не має BYOK — команда прив'язана до моделей і цінової політики
Anthropic. Тому варто стежити за: (а) зміною тарифів/rate-лімітів підписок
Pro/Max та API, які прямо впливають на вартість роботи; (б) вимогами data
retention для окремих моделей (напр. несумісність Fable 5 із ZDR), що може
зачепити compliance. Мітигація — тримати `AGENTS.md` крос-tool (Task C) і
зберігати Junie як реальну альтернативу, щоб перемикання між інструментами
лишалося дешевим.
