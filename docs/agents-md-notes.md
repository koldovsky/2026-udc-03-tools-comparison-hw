# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

Оригінал був однорядковим, tool-specific файлом під назвою «Cursor Rules» із
загальними побажаннями («use TypeScript», «keep things simple»). Узагальнив
його до крос-tool baseline:

- Прибрав прив'язку до конкретного інструменту: заголовок `# Cursor Rules` →
  `# AGENTS.md — app/`, додав явне формулювання «Applies to any AI coding
  tool working in this directory».
- Додав секцію **Stack**: TypeScript (ES2022, ESNext modules, strict mode),
  Node.js, `type: module` (ESM only), Vitest, без фреймворку й runtime-залежностей.
- Додав секцію **Commands**: `npm test` / `npm run test:watch` / `npm run
  typecheck`; explicit `build: не налаштовано` та `lint: не налаштовано`
  замість вигаданих команд, яких немає в `package.json`.
- Розписав загальні побажання («clean code», «functional style») у 5
  конкретних конвенцій: чисті/малі функції, тільки named exports, явні `.js`
  розширення в імпортах (вимога ESM), колокейтед Vitest-тести з
  `describe`/`it` по функціях, коментарі лише для неочевидної логіки.
- Додав секцію **Guardrails**: незмінність публічних сигнатур `slugify` /
  `truncate` / `parseTags`, заборона ослаблення тестів, заборона нових
  залежностей/toolchain без прямого запиту, обов'язкове пояснення змін.

## Результат

**Cursor:** на промпт «опиши конвенції цього проєкту» прочитав кореневий
`AGENTS.md`, `docs/walkthrough.md`, `app/AGENTS.md`, `app/package.json` і
`app/src/text-utils.ts` без додаткових підказок. Відповідь структурована на
два рівні: конвенції репо (шляхи артефактів, guardrails домашки, перевірка
`npm test`) і конвенції `app/` (стек, команди, 5 правил коду, guardrails для
функцій). Усі деталі — «lint не налаштовано», явні `.js`-імпорти, named
exports, Vitest `describe`/`it`, заборона змінювати сигнатури `slugify` /
`truncate` / `parseTags` — взято з `AGENTS.md`, нічого не вигадано. На
уточнення «для app/» одразу звузив відповідь до `app/`-конвенцій без
повторного читання файлів і без додаткових нагадувань.

**Claude Code:** прочитав `CLAUDE.md`, кореневий `AGENTS.md` та `app/AGENTS.md`
разом з `app/package.json` без додаткових підказок і видав структурований опис
— контекст завдання, обов'язкові шляхи для артефактів (`docs/tool-matrix.md`,
`docs/comparison.md` тощо), стек і команди `app/`, 5 конвенцій коду, guardrails
і крок «як перевірити перед PR». Жодна деталь (наприклад, «lint не
налаштовано» чи заборона змінювати публічні сигнатури) не була вигадана —
все співпадає дослівно зі змістом `AGENTS.md`-файлів. Додаткових нагадувань
не знадобилось.

## Висновок

Для Cursor і Claude Code одного узагальненого `app/AGENTS.md` (+ кореневого
`AGENTS.md` з контекстом домашки) достатньо як крос-tool baseline: обидва
інструменти самостійно знаходять і підхоплюють файли без додаткових
налаштувань і дотримуються конвенцій без повторних нагадувань. Cursor
додатково підвантажує кореневий `AGENTS.md` як постійне workspace rule, тому
на загальний промпт відповідає ширше (репо + `app/`), але коректно звужує
фокус за уточненням. Tool-specific файли (`.cursor/rules`, `CLAUDE.md`) не
обов'язкові — достатньо посилання на `AGENTS.md`, як у `CLAUDE.md`.
