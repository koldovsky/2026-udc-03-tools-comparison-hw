# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

- Прибрано прив'язку до однієї IDE: заголовок «Cursor Rules» замінено на
  крос-tool «AGENTS.md — інструкції для будь-якого AI-агента».
- Уточнено стек: TypeScript 5.6 (strict), ESM, Vitest 2.x, Node.js 22+,
  бібліотека без runtime-залежностей.
- Додано секцію «Команди»: build відсутній — замість нього
  `npm run typecheck` (`tsc --noEmit`); `npm test` / `npm run test:watch`;
  лінтер **явно позначено як не налаштований** (щоб агент не вигадував команду
  і не додавав ESLint самовільно).
- Додано 5 конвенцій коду: іменовані `export function` без default-експортів
  і класів; чисті функції без side effects; явні типи в публічних сигнатурах;
  ESM-імпорти з розширенням `.js`; коментарі лише для неочевидного.
- Додано секцію «Тестування»: структура `describe`/`it`, обов'язковий
  регресійний тест для баг-фіксів, зелений `npm test` перед завершенням.
- Додано guardrails: не змінювати публічні сигнатури, не додавати залежності,
  не чіпати файли поза `app/`, не комітити самостійно, без секретів/PII.

## Результат

Усі три інструменти знайшли `app/AGENTS.md` самостійно, без згадки
про існування файла у промпті, і без нагадувань відтворили всі ключові пункти:
`npm run typecheck` замість build, «лінтер не налаштований — не додавати
ESLint», іменовані експорти без default/класів, обов'язкові регресійні тести
для баг-фіксів, заборону нових залежностей і самостійних комітів. Cursor дав
найдетальнішу відповідь (додатково звірився з `package.json`/`tsconfig.json`),
Codex — найлаконічнішу, але повну.

## Висновок

Одного узагальненого `AGENTS.md` виявилось достатньо як крос-tool baseline:
усі три інструменти підхопили його без tool-specific файлів і додаткових
налаштувань. Tool-specific файли (`.cursor/rules`, `CLAUDE.md`) лишаються
корисними хіба що як тонкі надбудови (наприклад, кореневий `CLAUDE.md` тут —
просто посилання на `AGENTS.md`), а не як місце для дублювання правил.
