# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

- Замінено заголовок «Cursor Rules» на tool-agnostic **AGENTS.md — `app/`** з явним зазначенням, що файл підходить для Cursor, JetBrains AI Chat / ACP, Claude Code тощо.
- Додано секцію **Стек**: TypeScript 5.6, strict mode, ESM, Vitest; окремої збірки немає.
- Додано секцію **Команди** з таблицею `npm test`, `test:watch`, `typecheck`; явно вказано, що **`build` і `lint` не налаштовано** (без вигаданих скриптів).
- Додано секцію **Структура** (`src/text-utils.ts` + `*.test.ts` поруч) і правило для нових утиліт (named export).
- Додано **5 конвенцій коду**: функціональний стиль, стиль файлу, `.js` в імпортах тестів, структура describe/it, незмінний публічний API.
- Додано **Guardrails** (тести, залежності, секрети, коментарі, Windows `nul`) і секцію **Як працювати з задачами** (тікети в `materials/`, acceptance criteria, регресійні тести).

## Результат

**Cursor IDE** перелічив конвенції `app/` у форматі, що відповідає узагальненому `AGENTS.md`: TypeScript 5.6 + ESM + Vitest; команди з каталогу `app/`; явно — `npm run build` / `npm run lint` не налаштовано; функціональний стиль, подвійні лапки, 2 пробіли, `camelCase`, імпорти з `.js`, один `describe` на функцію, незмінні сигнатури `slugify`/`truncate`/`parseTags`; guardrails (тести, deps, секрети, Windows `nul`); тікети в `materials/` з регресійними тестами.

**IntelliJ IDEA + Cursor (ACP)** дав майже ідентичну відповідь по суті: той самий стек, команди (`npm test` обов'язково зелений перед завершенням), дерево структури `app/src/`, ті самі 5 конвенцій і guardrails. Відмінності: трохи детальніший опис структури каталогів; додатковий абзац про homework-репо (кореневий `AGENTS.md`, шляхи `docs/*`, `cd app && npm test` перед PR) — ймовірно, через ширший контекст IDE/репо, а не через помилку в `app/AGENTS.md`.

Обидва інструменти **не вигадали** неіснуючих команд lint/build і **не повернулися** до старих «Cursor Rules».

## Висновок

Один узагальнений `app/AGENTS.md` достатній як крос-tool baseline для Cursor IDE і IntelliJ + Cursor (ACP): обидва інструменти без додаткових налаштувань відтворили ключові конвенції з одного промпта. IntelliJ-сесія схильна додавати контекст усього репозиторію (кореневий `AGENTS.md`), тому для чистого scope `app/` варто в промпті уточнювати «конвенції проєкту `app/`». Окремі tool-specific файли (`.cursor/rules`, JetBrains guidelines) для цієї задачі не знадобились.
