# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

- Прибрав прив'язку до одного інструменту: заголовок і preamble змінено з
  «Cursor Rules» на «AGENTS.md» з явним формулюванням «baseline context for
  any AI coding tool (IDE plugin, agentic IDE, or CLI agent)» — тонкий,
  tool-specific файл став крос-tool baseline.
- Додав секцію «Stack»: TypeScript (ES2022, ESM, `strict` +
  `noUncheckedIndexedAccess`), Vitest, без runtime-залежностей.
- Додав секцію «Команди»: `npm install`, `npm test` / `npm run test:watch`,
  `npm run typecheck`, і явно зазначив, чого **немає** — build-кроку і
  лінтера («not configured»), замість вигадувати команди, яких не існує.
- Розписав розмиті пункти («write clean code», «add comments for anything
  non-obvious») у 5 конкретних конвенцій: функціональний стиль без
  класів/default exports, збіг зі стилем файлу, коментарі лише для
  неочевидної логіки, малі однопризначені функції (з прив'язкою до
  `slugify`/`truncate`/`parseTags` як зразка), і структура тестів
  (`describe` на функцію, `it` на поведінку, `*.test.ts` поряд із source).
- Додав секцію «Guardrails», якої раніше не було явно: обов'язковий
  `npm test` після змін, заборона міняти публічні сигнатури без явного
  запиту, заборона додавати залежності без потреби, вимога пояснювати
  зміни поведінки й додавати тест, заборона `git commit`/`push` без
  дозволу користувача.

## Результат

Обидва інструменти самостійно знайшли й прочитали `app/AGENTS.md` без
жодного нагадування чи явного посилання на файл і відтворили всі його
секції коректно: stack, команди (включно з чесним «lint не налаштовано»),
5 конвенцій коду, guardrails. Розбіжностей у змісті немає. Cursor пішов
трохи далі — сам підтягнув конкретні цитати коду (`slugify`, `describe`-блок
тесту) як приклади, чого Claude Code у своїй відповіді не робив (обмежився
переказом пунктів файлу). Обидва інструменти чітко дотрималися формату
(заголовки/списки) і жоден не запитав уточнень.

## Висновок

Одного узагальненого `AGENTS.md` достатньо як крос-tool baseline для цієї
задачі: обидва інструменти підхопили його автоматично, без tool-specific
файлів чи додаткових налаштувань, і дали практично ідентичний за змістом
результат. Єдина відмінність — глибина ілюстрації (Cursor додав code-цитати
самостійно), що радше плюс до якості відповіді, а не ознака того, що
`AGENTS.md` інтерпретувався по-різному.
