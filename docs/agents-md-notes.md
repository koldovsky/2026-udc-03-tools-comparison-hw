> питанням — див. "Висновок" нижче.

## Що додано/змінено в `app/AGENTS.md`

- Прибрано прив'язку до одного інструменту: заголовок був `# Cursor Rules`
  (Cursor-специфічний файл `.mdc`-стилю) → тепер `# AGENTS.md — app/`,
  нейтральний щодо інструменту, з явним реченням "should work the same
  whether it's read by Claude Code, Cursor, GitHub Copilot...".
- Додано секцію **Stack**: TypeScript, ESM, Vitest, Node 22+ — цього раніше
  не було взагалі (старий файл лише казав "small TypeScript project").
- Додано секцію **Команди**: `npm install` / `npm test` / `npm run
  test:watch` / `npm run typecheck`. **Lint явно позначено як
  "не налаштовано"** — у репозиторії немає ESLint/Prettier конфігурації
  (перевірено: `.eslintrc*` відсутній), тож замість вигаданої lint-команди
  файл чесно вказує typecheck як найближчий доступний static-check і
  просить задокументувати lint тут, якщо його колись додадуть. Build також
  позначено як "не застосовується" (немає emit-кроку).
- Додано 5 конкретних конвенцій коду (чисті функції без side-effects, тест
  на кожен export, named exports only, не змінювати публічні сигнатури без
  явного запиту, читабельність важливіша за стислість) — раніше було 4
  дуже загальні пункти ("write clean code", "follow existing style").
- Додано секцію **Guardrails**: не ламати існуючі тести без явної вказівки
  (а якщо потрібно — свідомо оновити тест і сказати про це), завжди
  прогонять `npm test` перед тим, як вважати задачу завершеною, без
  секретів/PII, не робити зайвих рефакторів поза скоупом задачі.

## Результат

Claude Code (реальний прогін): дав повну, точну відповідь — стек (TypeScript
ESM, Vitest, Node 22+, без фреймворка/бандлера), команди (`npm install`,
`npm test`, `npm run test:watch`, `npm run typecheck`, явно зазначив
відсутність lint), 5 конвенцій коду і 5 guardrails — усе дослівно відповідає
вмісту `AGENTS.md`, без домислів і без потреби додатково нагадувати про
файл (файл був єдиним джерелом контексту, яке агенту дали в промпті).

Другий інструмент: результату немає (див. таблицю вище) — це навмисно
залишено порожнім, а не заповнено вигаданими даними.

## Висновок

Для одного, реально перевіреного інструменту (Claude Code) `AGENTS.md` як
крос-tool baseline спрацював без жодних застережень: агент дотримався його
дослівно без додаткових нагадувань. Але **ця вправа за своєю суттю вимагає
доступу до 2+ фізично різних інструментів**, і жоден один CLI-агент (навіть
запущений як кілька незалежних підпроцесів) не може чесно відтворити другий
рядок таблиці — Claude Code, що запускає сам себе вдруге, не є незалежною
перевіркою в іншому інструменті. Інструкція в `walkthrough.md` ("перевірте
підхоплення в мінімум 2 інструментах") сама по собі зрозуміла й
однозначна для людини з двома встановленими інструментами — проблема
виникає тільки в сценарії "один агент виконує домашнє завдання сам за
себе", який сама вправа не передбачає і не повинна передбачати.
