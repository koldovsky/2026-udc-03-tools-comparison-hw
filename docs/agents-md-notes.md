# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

- **Перейменовано** заголовок з `# Cursor Rules` на `# AGENTS.md — app/` — прибрана прив'язка до конкретного інструменту.
- **Додано секцію «Стек»**: TypeScript strict mode, Node 22+, Vitest — щоб будь-який інструмент одразу знав технологічне оточення.
- **Додано секцію «Команди»**: `npm test` як єдина актуальна команда;
- **Розширено конвенції** з двох розмитих рядків («write clean code») до 5 конкретних: named exports, функціональний стиль, без коментарів, тести поруч із кодом, один концепт на функцію.
- **Додано секцію «Guardrails»**: заборона змінювати публічні сигнатури трьох функцій, заборона нових залежностей без запиту, заборона комітити секрети.

## Промпт верифікації

> "Add a new util reverseWords(input: string): string following the pattern of the existing functions in text-utils.ts. Add a test. No explanation — just do it."

## Результат

**Claude Code** — додав `reverseWords` як named export у кінці `text-utils.ts`, без default export, без коментарів у тілі функції, тест поклав у `text-utils.test.ts` поруч із файлом. Сам запустив `npm test`, переконався що зелено. Усі 5 конвенцій з `AGENTS.md` дотримані без жодного нагадування.

**Cursor** — після явного `@AGENTS.md` в промпті також додав named export і тест поруч із кодом. Але додав однорядковий коментар `// reverses word order` до функції — це суперечить конвенції «без коментарів». Тобто Cursor підхопив структурні конвенції (exports, розташування тестів), але стилістичні нюанси («без коментарів») засвоїв гірше.

## Висновок

Один `AGENTS.md` працює як спільний baseline, але повністю «без налаштувань» він підхоплюється тільки в інструментах, що його шукають автоматично. У Cursor потрібен явний `@`-референс або дублювання в `.cursor/rules`. Для команди, що використовує кілька інструментів, найнадійніший варіант — тримати `AGENTS.md` як джерело правди і автоматично синхронізувати його з tool-specific файлами, а не вести їх окремо.
