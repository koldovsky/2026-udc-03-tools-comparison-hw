- Прибрав заголовок і формулювання "Cursor Rules" — файл був явно
  заточений під один інструмент; замінив на нейтральний `# AGENTS.md`.
- Додав секцію **Stack**: TypeScript ES2022/ESM/`strict`, Node 22+, Vitest,
  без runtime-залежностей.
- Додав секцію **Commands**: `npm test` / `npm run test:watch`,
  `npm run typecheck`, build — немає (пакет споживається як TS-джерело),
  lint — явно написано "не налаштовано" (а не вигадана команда).

  конкретних конвенцій: чисті малі функції без I/O, явні типи параметрів/
  повернення на експортованих функціях, functional style де доречно,
  збіг зі стилем файлу, коментарі лише для неочевидного.
- Додав окрему секцію **Guardrails**: не міняти публічну сигнатуру
  `slugify`/`truncate`/`parseTags`, не додавати залежності без причини,
  не ламати тести, додавати регресійний тест при фіксі бага, пояснювати
  зміни.

## Висновок

Одного `AGENTS.md` достатньо як крос-tool baseline — обидва інструменти
підхопили й дотримались його без tool-specific конфігурації. Різниця не в
дотриманні правил, а в ширині контексту: Cursor за замовчуванням підняв
увесь репозиторій (кореневий `AGENTS.md`, walkthrough, темплейти), тоді як
Claude Code (у цьому прогоні) обмежився директорією задачі — тобто для
Claude Code варто явно вказувати робочу директорію/scope, якщо потрібен
контекст усього репо, а не покладатись на автоматичне підняття "нагору".
