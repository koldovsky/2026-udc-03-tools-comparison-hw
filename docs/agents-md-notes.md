# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

Було: тонкий, «заточений під один інструмент» файл (специфічні фрази на кшталт
«починай відповідь з 🧠», «показуй пропозиції по одній», фейковий стек
React 19 / app router, вигадані команди `dev`/`build`/`lint`).

Стало: узагальнений крос-tool baseline, більше не прив'язаний до жодного
конкретного інструмента. Явно зазначено, що файл читається будь-яким асистентом
(agentic IDE, IDE-плагін або CLI-агент).

Секції у новому файлі:

- **Project** — короткий опис sample-модуля (`text-utils.ts`), grounded у
  реальному коді, а не у вигаданому стеку.
- **Stack** — реальний стек із `package.json`/`tsconfig.json`: TypeScript `^5.6`
  (strict + `noUncheckedIndexedAccess`), ESM, ES2022, Node 20.9+, vitest.
- **Commands** — виправлено зламаний рядок `dev/test/build/lint>`. Тепер по
  одній команді на рядок: `test`, `test:watch`, `typecheck`; для
  `build`/`dev`/`lint` чесно вказано **not configured** замість вигаданих
  команд (як вимагає Task C — «lint: не налаштовано»).
- **Conventions** — 6 конвенцій коду (чисті функції, функціональний стиль,
  явні return-типи, форматування 2 пробіли/подвійні лапки/крапки з комою,
  коментарі лише для неочевидного, обовʼязкове vitest-покриття edge cases).
- **Guardrails** — не міняти публічні сигнатури `slugify`/`truncate`/`parseTags`,
  без нових runtime-залежностей, лишатися в межах стеку, не комітити секрети,
  Windows/Git Bash пастка з `2>nul`.
- **How to verify** — `npm test` зелений + `npm run typecheck` без помилок.

## Результат

GiHub Copilot/Raptor Mini - спочатку не знаходив другий файл, потім після уточнюючого питання знайшов 
і хотів використовувати зведені правила. Після уточнення що потрібно використовувати тільки правила проекту,
почав їх використовувати безе перевірки.
Claude Code/Opus 4.8 - відразу знайшов два файли, запропонував зробити узагальнюючий файл на основі обох файлів, 
також перевірив чи праюють команди і додав коментарі.
