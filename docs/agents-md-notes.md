## Результат

**Claude Code** (перевірено на оптимізованій версії `app/AGENTS.md`):
- Читає файл самостійно перед кодуванням
- Процитує конвенції дослівно й дотримується їх без нагадувань
- Виконує `npm run typecheck` і `npm test` автоматично
- Не додає лінтери, пославшись на guardrail

**GitHub Copilot (Chat)** (перевірено на оптимізованій версії `app/AGENTS.md`):
- Генерує функції й тести, дотримуючись Good/Bad прикладів
- Краще розуміє конвенції завдяки таблицям й явним ❌/✓ символам
- Запускає `npm test` після підказки (очікувано для completion-інструменту)
- Не пропонує eslint/prettier без запиту (завдяки секції Common Mistakes)

**JetBrains AI Assistant + Junie** (перевірено на оптимізованій версії `app/AGENTS.md`):
- Читає та враховує `app/AGENTS.md` автоматично завдяки індексації IDE
- Junie запускає перевірки в IDE: `npm run typecheck`, `npm test`, показує логи
- Дотримується правил: без нових залежностей/лінтерів, зберігає сигнатури функцій
- Завдяки семантичному індексу впевненіше тримає крос-файловий контекст

**Висновок (коротко):** Узагальнений, максимально явний `AGENTS.md` однаково добре
працює для повноцінних агентів (Claude Code), IDE‑агентів (JetBrains AI Assistant + Junie)

Common Mistakes — окремі tool‑specific варіанти не потрібні.

## Висновок

- Один узагальнений `AGENTS.md` достатній для всіх інструментів; зміни не потребують
  tool‑specific файлів.
- Критичні елементи: **таблиці для сканування**, **Good/Bad приклади коду**, **секція
  Common Mistakes**, **явні ❌/✓ маркери** та **explicit Verification Checklist**.
- Без цих елементів completion‑інструменти (Copilot) часто потребують додаткових
  нагадувань; із ними — дотримуються конвенцій автоматично.
- Повноцінні агенти (Claude Code) та IDE‑агенти (JetBrains AI Assistant + Junie)
  зазвичай дотримуються конвенцій і без додаткових підказок, а явні таблиці й приклади
  підвищують передбачуваність і швидкість для всіх.
