# Нотатки до AGENTS.md (Task C)

## Що додано/змінено в `app/AGENTS.md`

- Прибрано прив'язку до одного інструменту: заголовок «Cursor Rules» замінено
  на нейтральний «AGENTS.md» із явним адресатом — «будь-який AI coding agent
  (IDE-плагін, agentic IDE або CLI-агент)».
- Додано секцію **Project** — що це за пакет (чисті текстові хелпери,
  спільна база для вправи порівняння інструментів).
- Додано секцію **Stack**: TypeScript 5 (strict, ESM), Node 22+, Vitest із
  колокованими тестами.
- Додано секцію **Commands**: `npm test`, `npm run test:watch`,
  `npm run typecheck`; чесно зафіксовано, що **lint і build не налаштовані**
  (з явною забороною вигадувати неіснуючі команди).
- Розмиті поради («write clean, readable code») замінено на 5 конкретних
  конвенцій: чисті функції з іменованими експортами, явні типи
  параметрів/повернення, 2 пробіли + подвійні лапки + крапки з комою,
  ланцюжки `map`/`filter`/regex замість імперативних циклів, тести `*.test.ts`
  поруч із кодом (Vitest `describe`/`it`/`expect`).
- Додано **Guardrails**: не міняти публічні сигнатури `slugify`/`truncate`/
  `parseTags`, кожен баг-фікс — із регресійним тестом, `npm test` перед
  завершенням, ніяких секретів/`.env`.

## Результат

**Claude Code:** відповідь — фактично структурований переказ оновленого
`app/AGENTS.md`: стек, чотири команди, всі конвенції стилю та guardrails,
включно з нюансом «lint/build навмисно не налаштовані». Це підтверджує, що
файл підхоплено автоматично (у промпті файл не згадувався), причому саме
nested-версія з `app/`.

**Cursor:** структурований переказ `app/AGENTS.md` без згадки файлу в промпті:

- **Проєкт:** крихітна TS-бібліотека чистих текстових хелперів (`slugify`,
  `truncate`, `parseTags`) у `src/text-utils.ts`; спільна база для вправи
  порівняння — зміни мінімальні й сфокусовані.
- **Стек:** TypeScript 5 (strict, ESM), Node 22+, Vitest із колокованими
  тестами `src/*.test.ts`.
- **Команди** (з `app/`): `npm test`, `npm run test:watch`,
  `npm run typecheck`; lint і build **навмисно не налаштовані** — не
  вигадувати неіснуючі команди.
- **Конвенції:** чисті функції з іменованими експортами; явні типи
  параметрів/повернення; 2 пробіли, подвійні лапки, крапки з комою;
  ланцюжки `map`/`filter`/regex замість імперативних циклів; тести поруч із
  кодом, Vitest `describe`/`it`/`expect`, групування по функції.
- **Guardrails:** не міняти публічні сигнатури `slugify`/`truncate`/`parseTags`;
  кожен баг-фікс — із регресійним тестом; `npm test` перед завершенням;
  без секретів/`.env`; пояснювати зміни.

Підтверджує автоматичне підхоплення nested `AGENTS.md` через вбудовану
підтримку Cursor (без `.cursor/rules` дублікату для цих конвенцій).

## Висновок

Одного узагальненого `app/AGENTS.md` достатньо як крос-tool baseline для
Cursor і Claude Code: обидва інструменти підхопили його автоматично в свіжій
сесії, без згадки файлу в промпті верифікації, і відтворили ті самі секції —
стек, реальні команди, чесне «lint/build не налаштовані», конкретні конвенції
коду та guardrails. Окремі tool-specific файли (`.cursor/rules`, `CLAUDE.md`
в `app/`) для цих правил не знадобилися.

Різниця лише в шарі контексту: Cursor додатково бачить кореневий `AGENTS.md`
репо й user rules, Claude Code — лише nested-файл у робочій директорії `app/`.
Це не заважає дотриманню конвенцій, але в Cursor варто писати `app/AGENTS.md`
самодостатнім, щоб він не залежав від батьківського контексту. Tool-specific
файли лишаються потрібними для можливостей поза спільним знаменником
(skills, hooks, slash-команди, MCP-конфіг конкретного інструмента).
