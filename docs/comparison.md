Усі три інструменти зійшлись на одному й тому самому виправленні:

```ts
if (suffix.length >= maxLength) {
  return suffix.slice(0, maxLength);
}
return input.slice(0, maxLength - suffix.length) + suffix;
```

## Висновок

На цій конкретній задачі всі три інструменти впорались **з першого промпту без
жодної додаткової ітерації**: кожен знайшов справжню причину (суфікс додавався
поза лімітом `maxLength`), застосував ідентичний за логікою фікс, додав обидва
необхідні регресійні тести і лишив 8/8 тестів зеленими. Різниця у швидкості
виявилась несуттєвою (39–45 секунд). Здивувало, наскільки однаковим вийшов
фінальний код — аж до однакової структури умов. Відмінності проявились не в
якості коду, а в UX та прозорості витрат: Codex дає найточнішу статистику
(`/status`), Cursor — найнаочніший огляд контексту і безпечний worktree, Claude
Code — найповніший наскрізний робочий процес у терміналі. Фінальний фікс у репо
залишено від Claude Code (функціонально еквівалентний усім трьом).
