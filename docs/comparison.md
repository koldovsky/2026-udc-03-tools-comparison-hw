## Висновок

На цій конкретній задачі обидва інструменти впорались однаково добре за
результатом: обидва знайшли справжню причину бага (не симптом) за один
промпт, без уточнюючих запитань, і згенерували практично ідентичний
чотирирядковий фікс у `truncate()` плюс пару регресійних тестів. Різниця
виявилась не в якості рішення, а в середовищі виконання — Cursor-агент
наштовхнувся на обмеження sandbox при запуску `npm test` і мусив повторити
спробу поза ним, тоді як Claude Code відпрацював без такого тертя. Токени/
вартість жоден інструмент не показав явно в цьому прогоні, тому за цим
критерієм порівняти не вдалось. Що здивувало: на такій чітко сформульованій
задачі (тікет уже містив псевдокод очікуваної поведінки) інструмент
практично не впливає на якість фіксу — відмінність буде помітнішою на менш
формалізованих задачах.

## Фінальний фікс у коді

Залишено поточну версію в `app/src/text-utils.ts` і
`app/src/text-utils.test.ts` (ідентичний фікс від обох інструментів;
тестовий edge case узято у версії Cursor — там дві перевірки замість однієї,
трохи ширше покриття). `cd app && npm test` — зелений (8/8).
