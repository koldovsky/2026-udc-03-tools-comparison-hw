## Висновок

Усі п'ять прогонів (3 обов'язкові інструменти + 2 бонусні на тому самому CLI з
іншою моделлю) знайшли справжню причину бага з першого промпту й жодного разу
не сплутали симптом із коренем; регресій не було в жодному випадку. Найшвидшим
і найдешевшим виявився Claude Code CLI на Sonnet 5 (34с API, $0.24, без
жодного тертя), тоді як той самий інструмент на Opus 4.8 — найдорожчий
(2хв19с, $0.89) і єдиний, що не видав фікс одразу: скіл
`test-driven-development`.

> Застереження щодо часу: цифри в колонці «час до фіксу» виміряні в **різних
> базах** (API-час у Claude Code, wall-час в Antigravity/OpenCode,
> UI-показник «Worked for…» у Codex) і на **різних машинах/ОС** (Codex — на
> Windows). Тому «найшвидший = 34с» слід читати як **орієнтир напрямку**, а не
> як строго «яблука до яблук» замір: єдине, що тут коректно порівнювати
> напряму — вартість і час у межах **одного** CLI на різних моделях (Sonnet 5
> проти Opus 4.8), де база вимірювання однакова.

У цьому прогоні скіл `test-driven-development` змусив модель зупинитися й
попросити користувача самого визначити, як ділити `maxLength`, перш ніж
реалізувати рішення. Найбезпечніша обробка edge-case `maxLength <= 0` — в
Antigravity і Codex, де окрема гілка стоїть **першою, до будь-якої іншої
перевірки**; OpenCode теж виніс її окремо, але **другою** (після
`input.length <= maxLength`) — функціонально теж коректно, хоча й не «до
всього іншого». Обидва прогони Claude Code CLI цю гілку взагалі опустили,
залишивши вузьку, тікетом не покриту діру для від'ємного `maxLength` з
довгим suffix. Найбільше здивувало те, що вибір
моделі в межах одного інструменту (Sonnet 5 проти Opus 4.8) змінив вартість і
час сильніше, ніж сам вибір інструменту, а Codex самостійно обійшов реальне
обмеження Windows Group Policy на робочій машині, не питаючи користувача, як
діяти.

Фінальний фікс, залишений у `app/src/text-utils.ts` — від **Antigravity 2**: з
усіх аудитів саме тут найповніше покриття тестами (включно з явним тестом на
`maxLength <= 0`) при коректному й безпечному порядку перевірок. Регресійний
тест на сценарій із `materials/task-bug-fix.md` присутній (`BUG-101` у назві
теста), сигнатура `truncate(input, maxLength, suffix)` не змінена, `npm test`
зелений.

> Примітка: `npm test` наразі показує більше 9 тестів і додатковий файл
> `src/string-utils.ts` в `app/src/` — це не частина Task B. Це artifacts
> перевірки Task C (`wordCount()` від GitHub Copilot Free, `string-utils.ts`
> від Antigravity 2 — обидва додані під час перевірки підхоплення
> `AGENTS.md`, задокументовано в `docs/agents-md-notes.md`), залишені в коді
> навмисно як докази для рецензента. Регресія на `BUG-101` з цієї секції
> лишається в `truncate` без змін.
