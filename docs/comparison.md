## Висновок

Обидва інструменти впоралися з першого промпту (тікет вставлено «як є», без переказу) приблизно за 2 хвилини і знайшли **справжню причину**, а не симптом — суфікс не враховувався в межі `maxLength`. Обидва додали регресійний тест на сценарій із тікета та edge case `suffix.length >= maxLength`, і жоден не спричинив регресій. Головна відмінність — не якість фіксу (вона однакова), а середовище: Junie спирається на IDE-індекси JetBrains для навігації, тоді як Claude Code робить те саме агентно в терміналі (Grep/Read + `npm test`), що робить його придатним і поза IDE (CI, віддалений сервер). Здивувало, наскільки збіглися результати: на добре сформульованому тікеті з чіткими acceptance criteria різниця між інструментами звелась до UX, а не до правильності.
