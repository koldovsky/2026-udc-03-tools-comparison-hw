## Висновок

На цій конкретній задачі обидва інструменти знайшли справжню причину бага з першого промпту. Ключова різниця виявилась у кількості ручних кроків: Claude Code пройшов від тікета до успішних тестів повністю автономно, тоді як у Cursor довелось тричі натискати «Accept» і самостійно запустити `npm test`. Якщо задача коротка й одноразова — Cursor цілком комфортний. Але на задачах, де потрібно кілька ітерацій або перевірка тестів.

Що здивувало: очікував, що Cursor із вбудованим контекстом відкритого файлу спрацює швидше — але він все одно попросив уточнення щодо edge case, якого Claude Code вирішив самостійно
