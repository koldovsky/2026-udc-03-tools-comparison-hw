## Висновок

Обидва інструменти впорались з одного промпту й знайшли справжню причину
бага (`slice(0, maxLength)` не резервував місце під `suffix`), прийшовши
практично до однакового фіксу — без регресій і з повним проходженням
8 тестів. Claude Code був трохи швидшим і «чистішим» за кількістю кроків:
один прохід Read → Edit → `npm test` без повторних запусків. Cursor
відпрацював на ~30–60 секунд довше через TDD-підхід (спочатку дописав
failing-тести, лише потім фікс) і один повторний запуск тестів через
sandbox EPERM — по суті це friction інструменту, а не помилка агента.
Найбільше здивувало, наскільки однаковим вийшов сам фікс у обох
інструментів: за різного способу передачі тікета (paste у Claude Code
проти `@task-bug-fix.md` у Cursor) обидва самостійно підтягнули стиль
наявних тестів і жодного разу не перепитали постановку задачі.
