Обидва інструменти знайшли справжню кореневу причину бага (суфікс не враховується в межі зрізу) та досягли фіналу з 0 регресій і 8/8 регресійними тестами на задачу (20/20 при повній верифікації всіх функцій включно нові утиліти з Task C/E). **Sonnet 4.6** завершив за 1 чисту ітерацію: явно озвучив причину, написав компактний код та акуратні тести з прив'язкою до BUG-101. **Haiku 4.5** потребував 2 редагувань (спочатку не обробив edge case), але результат однаково якісний. **За вартістю:** Haiku $0.20 vs Sonnet $0.50 — Haiku дешевший у 2.5 раза при однаковому якісному результату на простому bug-fix. **Рекомендація:** для типових bug-fix завдань з чітко описаними edge cases (як у цьому тікеті) Haiku забезпечує достатню якість за мінімальну вартість; Sonnet корисний для складних рефакторів, де надійність за 1 хід критична.
