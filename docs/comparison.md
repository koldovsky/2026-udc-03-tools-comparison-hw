Задача: `materials/task-bug-fix.md` (та сама задача в усіх інструментах — для коректного порівняння «яблука з яблуками»).

| ----------- | --------------------: | -------------------------: | ------------------------------------------------------------------------------------------------------ | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Claude Code |                 ~5 хв | 1 промпт + 2 підтвердження | ✅ Знайшов справжню причину бага, додав регресійні тести, врахував edge cases, усі тести пройшли (8/8)  | Не відображалось             | Самостійно знайшов потрібні файли, запропонував TDD-підхід (спочатку тест, потім виправлення), сам обійшов проблему PowerShell через `npm.cmd`, після виправлення автоматично перевірив результат. |
| ChatGPT     |                 ~8 хв |                  2 промпти | ✅ Запропонував правильне виправлення та регресійний тест, пояснив причину бага, регресій не очікується | Не відображалось             | Добре пояснює логіку виправлення та допомагає з аналізом, але не має прямого доступу до локального workspace, тому користувач сам застосовує зміни та запускає тести.                              |

## Висновок

На цій задачі Claude Code впорався швидше завдяки агентному режиму роботи. Одного промпту було достатньо, щоб інструмент сам знайшов потрібні файли, додав регресійний тест, виправив реалізацію та перевірив результат запуском тестів. ChatGPT також правильно визначає причину проблеми та пропонує якісне рішення, але більше підходить як AI-асистент для аналізу й підготовки змін, тоді як виконання цих змін залишається за користувачем.
