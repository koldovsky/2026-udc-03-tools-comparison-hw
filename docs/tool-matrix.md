> **\* Рядок «Enterprise-функції» відповідає на питання: чи пройде інструмент перевірку безпеки у великій компанії?
> - **SOC 2 Type II** — незалежний аудит підтвердив, що вендор реально захищає дані (не разова перевірка, а спостереження за кілька місяців). Без цього сертифіката корпоративні закупівлі зазвичай не пропускають інструмент.
> - **SSO (SAML/SCIM)** — співробітники входять через корпоративний акаунт компанії (Okta, Azure AD), а не через особисті; коли людина звільняється, доступ забирається автоматично.
> - **Data residency** — гарантія, у якій країні обробляється код (важливо для GDPR та регульованих галузей).
> - **Audit logs** — журнал «хто, коли і що робив» для розслідувань і комплаєнсу.
> - **IP indemnity** (лише Copilot) — GitHub бере на себе юридичну відповідальність, якщо згенерований код порушить чиїсь авторські права.
> - **Self-hosted** (лише JetBrains) — моделі працюють на серверах самої компанії, код взагалі не потрапляє до вендора — найсильніша відповідь на вимоги ізоляції, сильніша за будь-який сертифікат.

## Висновок

Переможці за кожним критерієм матриці. Правило підрахунку: у кожному рядку рахується **рівно один** primary-переможець (по одному очку); якщо у критерія є сильний альтернативний варіант, він згаданий у колонці «Чому» як примітка, що в підсумковий рахунок не входить:

| Критерій                                     | Переможець (1 очко)      | Чому                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| --------------------------------------------- | ------------------------ |----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Вибір/якість моделі                          | **Claude Code** (якість) | Claude — найсильніші агентні моделі (Opus 4.8, Fable 5), але лише один вендор; JetBrains — єдиний з локальними моделями (Ollama/LM Studio) і формально перемагає за розмаїттям вибору моделей (примітка, не рахується в підсумку), проте мультипровайдерний вибір (GPT/Claude/Gemini) є й у Copilot, тож сам по собі він не унікальний                                                                                                                                                                                |
| Агентна автономність та multi-step execution | **Claude Code**          | Найглибша автономія: subagents, hooks, headless-режим; Junie та agent mode наздоганяють, але поступаються в терміналі та CI                                                                                                                                                                                                                                                                                                                                                                                           |

| Rules / commands / skills / MCP              | **Claude Code**          | Найбагатша розширюваність: skills + hooks + MCP + CLAUDE.md/AGENTS.md; у конкурентів є rules і MCP, але немає еквівалента hooks і кастомних skills                                                                                                                                                                                                                                                                                                                                                                    |

| Приватність / навчання на даних              | **Claude Code**          | Claude — найсильніша політика серед хмарних варіантів і застосовується до самого агента: без навчання на API-даних за замовчуванням + BYO-cloud (напр., Claude Code через власний AWS Bedrock акаунт компанії). JetBrains AI Assistant — найсильніша ізоляція (локальні моделі, напр. Qwen2.5-Coder або Llama через Ollama/LM Studio): код фізично не покидає машину, але це примітка, що не рахується як перемога рядка, бо **Junie (агент) локальних моделей не підтримує** — лише Junie CLI через кастомні профілі |

**Рахунок 4 : 2 : 1** (одне очко за рядок, рахується лише колонка «Переможець»; примітки в «Чому» на рахунок не впливають) — **Claude Code**: модель/якість, автономність, rules/skills/MCP, приватність; **GitHub Copilot**: ціна, enterprise; **JetBrains AI + Junie**: індексація кодової бази. Розрив помітний, але не означає «Claude однозначно переміг»: підсумок залежить від ваги критеріїв для конкретної команди — для enterprise-закупівлі вирішальними будуть два «копілотівські» рядки, а для проєкту з вимогою нульової мережевої передачі даних вирішальною стане примітка про локальні моделі JetBrains (рядок «Приватність»), а не сам рахунок.

Два виміри вибору, яких рахунок не показує:

- **Форм-фактор задачі.** Для нашого випадку (щоденна робота в IntelliJ + агентна задача «тікет → фікс → PR») найсуттєвішою виявилась відмінність «агент у терміналі vs плагін в IDE»: жоден інструмент не закриває обидва сценарії однаково добре, тому висновок — комбінувати primary/secondary інструменти для різних задач (**Claude Code** для агентної частини, **GitHub Copilot** для щоденного автодоповнення й навігації в IntelliJ).
- **Зрілість команди у використанні ШІ.** Команді з початковим рівнем (перший досвід з AI-асистентами) зручніший **GitHub Copilot**: інлайн-автодоповнення працює одразу «з коробки» і автоматизує більшість рутини без потреби вчити нові команди чи практики. Досвідченій команді, що вже вміє формулювати задачі для агента і перевіряти результат, більше дає **Claude Code** — менше автоматичних підказок, натомість повна автономія (subagents, hooks, headless-режим).

### Рекомендації

**Комбінуйте, а не обирайте один інструмент.** Пара «IDE-плагін + агент» (Copilot + Claude Code) закриває і щоденне автодоповнення, і автономні задачі.

**Хмарні/CI-агенти без участі розробника** — деталі, яких немає в жодній таблиці вище:
- **Claude Code** — web-додаток (claude.ai/code) для хмарних сесій, GitHub Action (`@claude` в issue/PR запускає агента в CI), Agent SDK для власних агентів на тій самій основі.
- **GitHub Copilot** — coding agent: призначаєте GitHub issue на Copilot, агент працює на GitHub Actions і відкриває draft PR (доступний з Pro+/Business).
- **JetBrains AI + Junie** — Junie CLI з BYOK для CI/CD-сценаріїв; хмарного агента формату «призначив issue — отримав PR» немає.

Саме ці хмарні/CI-агентні можливості між вендорами зближуються швидко — тут форм-фактор «CLI vs хмара vs CI» уже не головний критерій вибору. Але конкретно для нашого випадку (щоденна робота в IDE) вирішальним лишається інший розрив — відсутність інлайн-автодоповнення в Claude Code, — тому вибір усе ще визначають два критерії нижче.

**Критерій 1 — екосистема, в якій живе код:**

| Де живе код / команда                                              | Вибір                    | Чому                                                                                  |
|--------------------------------------------------------------------|--------------------------|---------------------------------------------------------------------------------------|
| GitHub (issues, PR, Actions)                                       | **GitHub Copilot**       | Coding agent дає найкоротший шлях від issue до draft PR без виходу з платформи        |
| Будь-який чи змішаний git-хостинг (GitLab, Bitbucket, self-hosted) | **Claude Code**          | Працює з будь-яким репозиторієм; хмара, IDE-розширення і CI не прив'язані до хостингу |
| IntelliJ-центрична команда                                         | **JetBrains AI + Junie** | Агент з доступом до семантичного індексу та інспекцій IDE                             |

**Критерій 2 — виміряна якість агента на власних задачах:** прогнати той самий тікет через кожного кандидата й порівняти час до робочого фіксу, кількість ітерацій та чи знайдена першопричина, а не симптом. Матриці порівняння відповідають лише на питання «що інструмент вміє»; на питання «наскільки добре» відповідає тільки такий прогін.
