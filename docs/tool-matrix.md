# Матриця можливостей AI-інструментів (Task A)

Інструменти, що порівнюєте: **Cursor** (розумний редактор коду на базі VS Code, з вбудованим AI) і **Claude Code** (AI-агент, що працює через командний рядок і теж має розширення для редакторів)

| Критерій | Cursor | Claude Code |
|---|---|---|
| Вибір/якість моделі | Можна обирати, яка AI-модель відповідає: Claude, GPT-5.x, Gemini, Grok — або довірити вибір інструменту (режим "Auto"). Плюс — гнучкість; мінус — якість відповіді залежить від того, яку модель обрано саме зараз. | Працює лише з моделями Claude (лінійка Sonnet/Opus/Haiku). Без вибору між компаніями-виробниками, зате поведінка стабільніша й передбачуваніша — інструмент і модель роблені однією командою. |
| Агентна автономність та multi-step execution (наскільки самостійно інструмент виконує складну задачу з кількох кроків) | Режим "Agent" сам змінює кілька файлів і запускає команди в терміналі; є "Background Agents" — задачі, що виконуються у фоні, поки користувач працює над іншим; є можливість відкотити крок, якщо результат не влаштував. | Побудований саме навколо самостійного виконання: читає файли, вносить зміни, запускає команди й тести, планує кроки наперед, може розбивати задачу між кількома паралельними "помічниками". Рівень дозволів налаштовується — від "питати перед кожним кроком" до "діяти повністю самостійно". Добре підходить для сценарію "видав задачу — отримав готовий результат" без відкритого редактора (саме це перевіряє Task E). |
| Індексація кодової бази (як інструмент "запам'ятовує" структуру проєкту, щоб швидко знаходити потрібний код) | Заздалегідь скановує весь проєкт і будує внутрішню карту коду при відкритті — завдяки цьому швидко знаходить потрібне за змістом запиту, а не лише за збігом слів. За замовчуванням ця карта будується на серверах Cursor (є режим приватності, щоб цього уникнути). | Готової карти немає — кожного разу, коли отримує задачу, сам переглядає файли й шукає потрібне (як людина, що відкриває папки одну за одною), плюс читає файл `CLAUDE.md` із описом проєкту. Менше налаштувань і завжди актуальна інформація, але на дуже великих проєктах такий пошук вимагає більше ресурсів AI-моделі. |
| Підтримка rules / commands / skills / MCP (файли-інструкції для AI, власні швидкі команди, підключувані "навички", підключення до зовнішніх сервісів) | Файли-інструкції (`.cursor/rules`), власні швидкі команди, підключення зовнішніх сервісів (MCP) через конфігураційний файл. Окремих підключуваних "навичок" немає — цю роль виконують файли-інструкції. | Файл-інструкція `CLAUDE.md` (можна мати кілька — для всього проєкту й окремо для підпапок), власні швидкі команди, підключувані "навички" (Skills), можливість розбивати роботу між кількома помічниками з різними ролями, автоматичні реакції на події (hooks), підключення до зовнішніх сервісів (MCP) в обидва боки. Найширший набір можливостей налаштування з двох. |
| Ціна та ліміти | Безкоштовний рівень з обмеженою кількістю запитів до AI; платний план ≈ $20/місяць з базовим обсягом, далі — доплата за фактичне використання; командний план ≈ $40/користувача/місяць. | Входить у підписку на Claude (базовий план ≈ $20/місяць з обмеженою кількістю повідомлень; розширений план $100–200/місяць з суттєво більшим лімітом) або оплата за фактичне використання через API. Інструмент сам показує, скільки коштувала кожна сесія — зручно для порівняння в Task B. |
| Enterprise-функції (можливості для великих компаній: єдиний вхід, сертифікати безпеки, контроль де зберігаються дані) | На командному плані: єдиний вхід через корпоративний акаунт (SSO), сертифікат безпеки SOC 2 Type II, примусовий режим приватності для всієї компанії, панель адміністратора, журнал дій користувачів. | На командному/корпоративному плані: єдиний вхід (SSO), керування користувачами (SCIM), журнал дій; сертифікат безпеки SOC 2 Type II; можливість обрати, у якому регіоні фізично зберігаються дані (через хмарних партнерів). |
| Приватність / політика навчання на даних | Режим приватності (увімкнений за замовчуванням на командному плані): код не зберігається на серверах компанії і не використовується для навчання AI-моделей. Без цього режиму код може тимчасово оброблятись компанією, що надає обрану модель. | Дані, передані через API, за замовчуванням не використовуються для навчання моделей (без окремої явної згоди користувача); для великих компаній доступний режим повного незбереження даних. Код не відправляється на побудову зовнішньої "карти проєкту", як у Cursor — менше даних покидає комп'ютер користувача за замовчуванням. |

## Висновок

Cursor зручніший людині, що працює в редакторі коду і хоче одразу бачити весь проєкт, з можливістю обрати AI-модель під конкретну задачу. Claude Code сильніший, коли задачу треба повністю доручити AI і отримати готовий результат без постійного нагляду — саме такий сценарій перевіряє Task E (задача → готовий Pull Request). Найбільша різниця для нашого випадку: спосіб роботи з проєктом (готова карта коду в Cursor проти самостійного пошуку в Claude Code щоразу заново) і приватність за замовчуванням (Claude Code передає менше даних назовні без явної згоди користувача). Можливості для великих компаній (єдиний вхід, сертифікати безпеки) в обох на прийнятному рівні — тут різниця не критична.
