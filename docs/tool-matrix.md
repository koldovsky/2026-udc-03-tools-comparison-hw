# Матриця можливостей AI-інструментів (Task A)

Інструменти, що порівнюються: **Claude Code** (Anthropic CLI) vs **Cursor Desktop** (Anysphere IDE)

| Критерій | Claude Code | Cursor Desktop |
|---|---|---|
| **Вибір/якість моделі** | Лише моделі Anthropic: Sonnet 4.6, Opus 4.8, Haiku 4.5. Контекстне вікно — до 1M токенів. Вибір моделі вручну або авто. Зовнішні провайдери не підтримуються. | Мульти-провайдерний: OpenAI (GPT-5.x), Anthropic (Claude Sonnet/Opus/Fable), Google (Gemini 3.x), xAI (Grok), власні моделі Composer 2.5 (агент) та Fusion (автодоповнення). Режим Auto або ручний вибір. |
| **Агентна автономність та multi-step execution** | Термінально-нативний агент першого класу: читає файли, редагує код, запускає shell-команди, git, тести, субагенти, фонові агенти, CI-інтеграція. Мінімальне ручне керування контекстом. Checkpoint/resume для довгих задач. | Агентний режим всередині редактора із «слайдером автономності» та inline review. Може виконувати термінальні команди й редагувати файли. Хмарні агенти доступні на Pro+. Підходить більше для парного програмування, ніж для повністю автономних задач. |
| **Індексація кодової бази** | Автоматично завантажує весь репозиторій у 1M-токенний контекст при старті. Не потребує окремого embedding-індексу — весь код доступний моделі безпосередньо. | Повна індексація репозиторію через embeddings для семантичного пошуку (@codebase). Код завантажується на сервер для обчислення ембедингів (plaintext видаляється одразу), але ембединги та метадані (хеші, імена файлів) зберігаються. Ефективний контекст ~70–120K токенів після внутрішнього скорочення. |
| **Підтримка rules / commands / skills / MCP** | CLAUDE.md для правил проєкту, кастомні /skills (slash-команди), hooks (pre/post-tool), MCP без ліміту на кількість серверів/інструментів, Tool Search для великих toolset-ів. Повна підтримка на всіх планах. | `.cursorrules` / Cursor Rules файли, командний маркетплейс для rules/skills/plugins, MCP підтримується на Pro+ і вище, хмарні агенти. Bugbot для автоматичних code review на Team-плані. |
| **Ціна та ліміти** | Pro $20/міс (~44k токенів / 5 год); Max $100/міс (5× Pro); Max 20× $200/міс. API pay-as-you-go: Sonnet 4.6 — $3/MTok вхід, $15/MTok вихід. Teams Premium (з Claude Code) — $100/місце/міс, мін. 5 місць. | Hobby — безкоштовно (обмежено); Pro $20/міс; Pro+ $60/міс (3×); Ultra $200/міс (20×). Teams: Standard $40/user/міс, Premium $120/user/міс (5× агент-ліміт). Enterprise — за домовленістю. |
| **Enterprise-функції (SSO/SOC2/data residency)** | SSO: SAML 2.0 і OIDC (Okta, Azure AD / Entra ID, Google Workspace, Auth0). SOC 2 Type I & II, ISO 27001:2022, ISO/IEC 42001:2023, BAA (HIPAA). Admin Console, Zero Data Retention (ZDR). | SSO: SAML/OIDC, SCIM для управління місцями. SOC 2 Type II (звіт — під NDA для Business). Аудит-логи, контроль моделей та репозиторіїв, сервісні акаунти, пулінг квот, пріоритетна підтримка. Self-hosting обмежений. |

Enterprise/Team — контрактне зобов'язання про невикористання даних для навчання. | Privacy Mode (увімкнено за замовчуванням у Business): дані не зберігаються постійно і не використовуються для навчання. Без Privacy Mode — Cursor може використовувати код, запити та дії для навчання моделей. DPA-угода у Business-плані. |

## Висновок

**Claude Code** виграє за рівнем агентної автономності (термінально-нативний, 1M-токенний контекст, subagents, CI), глибиною MCP-інтеграції та прозорістю privacy-політики (no-training за замовчуванням без додаткового налаштування). **Cursor Desktop** сильніший як щоденне середовище розробки завдяки мульти-модельному вибору, вбудованому редактору, семантичному @codebase-пошуку та зручному inline agent-режиму для парного програмування. Найбільша відмінність — у підході до контексту: Claude Code тримає весь репозиторій у єдиному вікні, тоді як Cursor покладається на embedding-пошук з ефективним вікном ~70–120K токенів. Оптимальна стратегія для команди — використовувати Cursor для поточної розробки з autocomplete і code review, а Claude Code для складних рефакторингів, автономних multi-step задач та CI-пайплайнів.
