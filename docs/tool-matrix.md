# Матриця можливостей AI-інструментів (Task A)

Інструменти, що порівнюються: **IntelliJ IDEA + Cursor (ACP)** та **Cursor IDE**

| Критерій | IntelliJ IDEA + Cursor (ACP) | Cursor IDE |
|---|---|---|
| Вибір/якість моделі | Той самий пул frontier-моделей, що й у підписці Cursor (OpenAI, Anthropic, Google, Cursor); перемикання в AI Chat. У ACP зараз один рівень reasoning на модель (напр. `gpt-5.3-codex` — лише `medium`). Немає Tab-autocomplete від Cursor — лише агентний чат. | Повний model picker + Auto mode (безліміт на платних планах). Tab completions на кожному рядку; Composer/Agent з вибором моделі та рівня reasoning. Cloud/Background Agents на Pro+. |
| Агентна автономність та multi-step execution | Агент через ACP: читає проєкт, редагує файли в редакторі IntelliJ, запускає команди в integrated terminal. Multi-step є, але UX — панель AI Chat, не нативний agentic workflow Cursor IDE. Немає Background/Cloud Agents, Bugbot, вбудованого diff-review як у standalone Cursor. | Agent mode за замовчуванням: multi-file edits, terminal, планування кроків. Background Agents (хмара) і Cloud Agents на Pro+. Інтегрований review змін, `@`-контекст, швидке застосування/відхилення diff у редакторі. |
| Індексація кодової бази | Семантичний пошук Cursor + глибока code intelligence JetBrains (рефакторинг, навігація, inspections, debugger). Сильна комбінація для Java/Kotlin/багатомовних монорепо, де важливі саме JetBrains-інструменти. Індексація Cursor працює через ACP-сесію — менш «вбудована», ніж у native IDE. | `@codebase`, embeddings, semantic search — нативно в UI. Дуже зручно для TS/JS/Python і швидкого огляду невеликих репо. Слабше за IntelliJ у важких Java-рефакторингах і enterprise-debugging, бо це VS Code fork, а не повноцінний JetBrains runtime. |
| Підтримка rules / commands / skills / MCP | MCP: проєктний `.cursor/mcp.json` (потрібно запускати `agent` з кореня проєкту й approve сервери; глобальна конфігурація MCP з Cursor IDE не завжди пробрасывається в ACP — відоме обмеження). Pro: $20/міс — extended Agent, MCP/skills/hooks, cloud agents. Pro+ $60, Ultra $200. Teams від $40/корист./міс (SSO, shared rules). Перевищення кредитного пулу — on-demand за API-ставками обраної моделі. |

## Висновок

Обидва варіанти використовують **той самий Cursor-агент і той самий billing**, але різняться «оболонкою»: IntelliJ IDEA + ACP виграє там, де критичні **рефакторинг, debugger і багатомовна IDE-екосистема JetBrains** (Java, Spring, Android тощо), тоді як **Cursor IDE** суттєво сильніший у **повноцінному agentic UX** — Tab, Composer, rules/skills/MCP без обмежень ACP, Background/Cloud Agents.

Найбільша практична різниця для щоденної роботи — **глибина інтеграції AI в workflow**: у standalone Cursor правила проєкту, MCP і швидкий цикл «промпт → diff → apply» працюють як єдина система; у IntelliJ агент потужний, але **rules/MCP і вторинні фічі Cursor передаються неповно**, а autocomplete залишається силою JetBrains, не Cursor.

Для TypeScript/невеликих репо (як цей homework) **Cursor IDE зручніший end-to-end**; для змішаного стеку з Java/Kotlin і звичною IntelliJ-середовищем **ACP — розумний компроміс** без другої підписки на AI, якщо готові миритися з обмеженнями інтеграції на рівні MCP і team rules.
