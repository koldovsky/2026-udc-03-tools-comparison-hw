# Матриця можливостей AI-інструментів (Task A)

Інструменти, що порівнюєте:  GitHub Copilot, Claude Code

| Критерій | GitHub Copilot  | Claude Code CLI |
|---|---|---|
| Вибір/якість моделі | GPT 5.4 /для знаходження багу в фронтенді знадобилось декілька ітерацій | Opus 4.6 для знаходження неточностей меньша кількість ітерацій, недолік більша вартість  |
| Агентна автономність та multi-step execution | присутня | присутня через Subagent |
| Індексація кодової бази | присутня кожні 60 секунд  | відсутня |
| Підтримка rules / commands / skills / MCP |  читає файл AGENTS.MD i CLAUDE.md, підтримує команди через папку promts, підтримує skils and mcp servers | тільки CLAUDE.md, підтримує команди, скіли і mcp servers|
| Ціна та ліміти | 10-15$, відсутня модель Opus | 20$  |
| Enterprise-функції (SSO/SOC2/data residency) | SSO/SOC2/data residency | SSO/SOC2 |
| Приватність / політика навчання на даних | навчання данних ввімкнене по дефолту для none-enterprise користувачів | навчання данних ввімкнене по дефолту для none-enterprise користувачів, не використовує інкогніто чати для тренування |

## Висновок

На мою думку в повсякденній роботі мені зручніше використовувати GitHub Copilot, оскільки він має сильнішу інтеграцію з засобами для розробки, підтримує використання декількох файлів правил, а також індексацію кодової бази. Також підтримка декількох типів моделей і можливість додавання нових, для мене є перевагою для покращення якості результату.
