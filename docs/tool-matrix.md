# Матриця можливостей AI-інструментів (Task A)

Інструменти: **Claude Code**, **Cursor**

| Критерій | Claude Code | Cursor |
|---|---|---|
| Вибір/якість моделі | claude-sonnet-4-6 за замовчуванням; перемикається на claude-opus-4-8 або claude-haiku-4-5 командою `/model`; доступ до всього Anthropic lineup | Claude 3.7 Sonnet / GPT-4o / Gemini 1.5 Pro — вибирається у налаштуваннях; модель легко змінити прямо в чаті |
| Агентна автономність та multi-step execution | Повноцінний агент: читає/пише файли, запускає shell-команди, прогоняє тести, відкриває PR — все в одному потоці без ручних підтверджень (якщо дозволено) | Agent mode: редагує кілька файлів, запускає термінал, але після кожного кроку чекає підтвердження; більш «supervised» |
| Індексація кодової бази | Файли читаються на запит через `Read`/`Bash`; векторного індексу немає — повна картина є лише якщо агент сам обійшов файли; CLAUDE.md дає статичний контекст | Вбудований семантичний індекс (embeddings усього репо); `@codebase` миттєво знаходить релевантний код навіть у великих проєктах |
| Підтримка rules / commands / skills / MCP | CLAUDE.md, `/skills`, hooks у settings.json, повноцінна підтримка MCP-серверів; можна задати будь-яку автоматизацію | `.cursor/rules` (раніше `.cursorrules`), User Rules, MCP-сервери підтримуються з 2025; набір опцій менший ніж у Claude Code |
| Ціна та ліміти | $20/міс Pro (включено, але є ліміти); необмежено через API за токенами (~$3/MTok для Sonnet); при активному агентному використанні витрати ростуть швидко | $20/міс Pro (500 «швидких» запитів/міс, далі slower); $40/міс Business; немає pay-per-token для кінцевого користувача |
| Enterprise-функції (SSO/SOC2/data residency) | Anthropic SOC 2 Type 2; SSO та SCIM на Enterprise-плані; data residency — поки лише US (EU-регіон в roadmap) | SOC 2 Type 2; SAML/OIDC SSO на Business+; вибір регіону (US/EU) для зберігання коду на Business-плані |
| Приватність / політика навчання на даних | Pro та API: дані не використовуються для тренування за замовчуванням; на Enterprise є BAA і zero data retention | Код не використовується для тренування на Pro+ («Privacy Mode» ввімкнено за замовчуванням); але метадані телеметрії збираються |

## Висновок

Якщо мені потрібно вибрати один інструмент для реальної щоденної роботи — я б поставив **Claude Code** на перше місце для задач, де потрібна справжня автономія: «візьми тікет і зроби PR» без моєї участі. Він єдиний із двох, хто реально може пройти весь шлях від читання задачі до відкриття PR самостійно. Платити за токени при активному використанні буде дорожче, ніж фіксований $20 Cursor, але контроль над моделлю та MCP-інтеграції це компенсують.

**Cursor** найкраще підходить як щоденний IDE-компаньйон: автодоповнення, inline-чат і агент — все в одному вікні, не виходячи з IDR.

Найсуттєвіша відмінність між двома інструментами виявилась у **агентній автономності**: Claude Code сам виконує задачу й звітує, Cursor зупиняється після кожного кроку й чекає підтвердження. На простих правках різниця мінімальна, але на складних multi-step задачах — це відчутно.
