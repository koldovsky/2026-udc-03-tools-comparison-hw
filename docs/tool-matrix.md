# Матриця можливостей AI-інструментів (Task A)

Інструменти, що порівнюємо: **Claude Code (CLI-агент)**, **GitHub Copilot (VS Code, agent mode)**, **Cursor (agentic IDE)**

| Критерій | Claude Code | GitHub Copilot (VS Code) | Cursor |
|---|---|---|---|
| Вибір/якість моделі | Claude Sonnet 4.5 / Opus 4.x / Haiku, перемикання прапорцем `--model` або командою `/model` всередині сесії | Picker моделей у Copilot Chat: GPT-4.1, GPT-4o, Claude Sonnet 4, Claude Opus 4.1, Gemini 2.5 Pro (залежить від плану); за замовчуванням "Auto" в agent mode | Власний picker: Claude Sonnet 4.5, GPT-5, Gemini 2.5 Pro, легка `cursor-small`; є "Auto"-режим, що сам вибирає модель за складністю запиту |
| Агентна автономність та multi-step execution | Повний агентний цикл read→edit→run→verify без відкритого редактора; headless `-p` режим для CI/скриптів; підзадачі через subagents | Agent mode виконує multi-file edits, запускає термінал, ітерує за помилками збірки/тестів; за замовчуванням запитує підтвердження на частину дій | Composer/Agent mode — multi-file edits із diff-переглядом прямо в редакторі, автозапуск команд у вбудованому терміналі; найглибша UI-інтеграція серед трьох |
| Індексація кодової бази | Немає окремого векторного індексу — покладається на grep/glob/read у реальному часі плюс `AGENTS.md`/`CLAUDE.md` як контекст-карту | Індексує workspace (embeddings) для `@workspace`-запитів; якість/свіжість індексу залежить від розміру репо, може відставати після великих правок | Вбудована інкрементна індексація кодової бази (codebase-wide embeddings) — історично найсильніша сторона Cursor для навігації у великому репо |
| Підтримка rules / commands / skills / MCP | `CLAUDE.md`/`AGENTS.md`, кастомні slash-commands (`.claude/commands`), Skills, повна підтримка MCP (stdio/SSE) | `.github/copilot-instructions.md`, підтримка `AGENTS.md` (з 2025), MCP-сервери в останніх версіях VS Code, custom "prompt files" | `.cursor/rules/*.mdc` (project rules) + підтримка `AGENTS.md`, MCP через `.cursor/mcp.json`; немає окремого "skills"-механізму |
| Ціна та ліміти | Pro $20/міс (ліміти повідомлень на 5-годинне вікно), Max $100–200/міс з вищими лімітами; API — окремо pay-as-you-go | Individual $10/міс, Business $19/user/міс; agent mode витрачає "premium requests" з місячної квоти, понад квоту — доплата за запит | Hobby (free, обмежено), Pro $20/міс (≈500 fast premium requests/міс, далі повільніше/дорожче), Business $40/user/міс |
| Enterprise-функції (SSO/SOC2/data residency) | Team/Enterprise плани Anthropic: SSO, SCIM, аудит-логи, SOC2 Type II; data residency обмежена (за замовчуванням US-based обробка) | Copilot Business/Enterprise: SSO через GitHub Enterprise, SOC2, org-level policy control (можна вимкнути matching публічного коду) | Cursor Business: SSO, примусовий privacy mode на рівні організації, SOC2 Type II |
| Приватність / політика навчання на даних | За замовчуванням дані API/Team не йдуть на тренування моделей (opt-out для API за замовчуванням) | Business/Enterprise: код не використовується для тренування публічних моделей; в Individual/free tier залежить від налаштувань акаунту | "Privacy mode" (опційно на Free/Pro, примусово на Business) — код не зберігається на серверах Cursor і не йде на тренування |

## Висновок

Найбільш відчутна відмінність для нашого сценарію — між "без вбудованого індексу, зате повний headless-контроль через файлову систему" (Claude Code) і "вбудований codebase-index із швидким semantic search прямо в IDE" (Cursor). Для CI/автоматизації (пор. Task E) CLI-агент явно виграє, бо не залежить від UI. Для швидких точкових правок у знайомому редакторі з живим diff-переглядом Cursor зручніший. Copilot має сенс там, де команда вже й так живе в GitHub Enterprise — SSO, procurement і security-review вже узгоджені на рівні організації, і це знижує організаційні витрати на впровадження навіть якщо "чистий" агентний UX слабший за Cursor.
