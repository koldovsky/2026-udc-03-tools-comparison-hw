## Висновок

Найсуттєвіша відмінність для нашого випадку — **модель індексації кодової бази**. Junie спирається на глибокі IDE-індекси JetBrains (символи, структура проєкту), що дає точний контекст «безкоштовно» всередині IDE, але прив'язує інструмент до екосистеми JetBrains. Claude Code натомість не будує персистентний індекс — він шукає агентно (Grep/Glob/Read) у будь-якому середовищі через термінал, що робить його переносимим (CI, віддалені сервери, будь-яка IDE), ціною того, що великий контекст доводиться реконструювати щоразу (частково компенсується prompt caching).

Друга ключова відмінність — **вибір моделі vs. якість агентного циклу**. Junie гнучкіший за моделями (BYOK для Anthropic/OpenAI/Google/xAI), тоді як Claude Code замкнений на моделях Anthropic, але за замовчуванням дає найсильнішу з них (Opus 4.8 на `xhigh`) плюс зрілі примітиви автономності — субагенти, детерміновані Workflows, hooks і фонові задачі. Для CLI-центричного, tool-to-PR сценарію цього ДЗ Claude Code виграє автономністю та переносимістю; для роботи, зосередженої в JetBrains IDE з потребою змішувати провайдерів моделей, Junie зручніший.
