# Матриця можливостей AI-інструментів (Task A)

У цьому документі я систематизував та порівняв можливості п'яти AI-інструментів розробки, з якими я працював як на робочих проєктах (з платними підписками на CLI-версії), так і у власних хобі-ініціативах:
1. **Claude Code** (CLI-інструмент від Anthropic; хобі — Pro-підписка, робота — API-білінг через корпоративний проксі)
2. **GitHub Copilot** (VS Code плагін, Free-план — хобі)
3. **Antigravity 2** (агентна desktop-платформа від Google: app + CLI + SDK; хобі, лише безкоштовні моделі)
4. **Codex CLI** (CLI-інструмент на базі моделей OpenAI, GPT-5.5; робота — API-білінг через корпоративний проксі)
5. **OpenCode** (open-source CLI-агент, мій основний інструмент на роботі — підключений через корпоративний проксі до GitHub Copilot та OpenAI)

Усі інструменти, якими я користуюсь на роботі, підключені через внутрішній корпоративний проксі (API-gateway) — тобто оплата йде за pay-per-token тарифами компанії, а не через особисті підписки. У хобі-проєктах, навпаки, використовую особисті підписки/free-тіри напряму.

---

## Спільні можливості (Common Features)

Перш ніж перейти до унікальних фіч, варто зазначити базовий рівень можливостей, який сьогодні мають практично всі сучасні AI-агенти:
* **Базова робота з файлами:** Читання, створення та редагування коду в межах робочої папки (workspace).
* **Базовий чат-інтерфейс:** Можливість ставити питання про код і отримувати пояснення без агентного виконання дій.

## Висновок

Для моєї повсякденної роботи найсильнішим інструментом виявився **Claude Code** — завдяки якості кодування моделей родини Claude та гнучкій системі hooks, яка додає детермінізму й безпеки в автоматизацію дій без роздування контексту. **GitHub Copilot** на Free-плані не дає вибору моделі (лише `Auto`) — це різко інша філософія за автономність порівняно з Claude Code/Antigravity 2, де я сам вирішую, якою моделлю й з якою «силою» (reasoning effort) виконати задачу; головна сила Copilot лишається в моментальному inline-автодоповненні, а не в агентних сценаріях. **Antigravity 2** зробив ставку на standalone-платформу з мульти-агентною оркестрацією та Scheduled Tasks — цікавий кандидат для комплексних рефакторів чи фонової автоматизації, хоча тепер прив'язаний до дорожчих тарифів Google AI Ultra. **Codex CLI** на GPT-5.5 з high reasoning effort у моєму досвіді відчувається менш автономним за Claude Code/Antigravity 2 в багатокрокових сценаріях. **OpenCode** — мій основний робочий інструмент з конкретної причини: на роботі Claude Code й Codex CLI теж ідуть через внутрішній корпоративний проксі (усі інструменти однаково «вміють» у проксі), але кожен з них все одно розмовляє лише зі «своєю» родиною моделей — Claude Code тільки з Claude, Codex CLI тільки з OpenAI. OpenCode, сконфігурований на той самий проксі, одним інтерфейсом покриває і OpenAI, і GitHub Copilot, тож економить не на самому проксі, а на кількості окремих вендор-специфічних CLI, які інакше довелося б тримати паралельно.

**Найсуттєвіша відмінність для мого випадку:** (1) консолідація навколо одного gateway — на роботі буквально все, включно з Claude Code й Codex CLI, вже рахується як pay-per-token через внутрішній корпоративний проксі, тож різниця не в тому, чи «вміє» інструмент у проксі (вміють усі), а в тому, що OpenCode одним інтерфейсом покриває і OpenAI, і GitHub Copilot, тоді як Claude Code/Codex CLI лишаються прив'язаними кожен до своєї моделі; (2) контроль над вибором моделі — Claude Code/Codex CLI/OpenCode дають явний вибір моделі й reasoning effort, тоді як мій Copilot Free звужений до непрозорого `Auto`; (3) глибина агентної автономності (Claude Code/Antigravity 2/OpenCode vs Copilot/Codex CLI); (4) наявність структурованого способу «навчити» інструмент конвенціям проєкту (Skills/hooks у Claude Code, AGENTS.md всюди) — це прямо впливає на те, скільки разів доводиться повторювати одні й ті самі інструкції.
