# CLI агент: тікет → PR (Task E, bonus)

Тікет: `materials/task-feature-ticket.md` (FEAT-102 — додати `capitalizeWords()`)

## CLI-агент

**Claude Code** (той самий CLI-агент, що виконував Claude-сторону Task B).

## Команда, яку запустили

Тікет переданий агенту як єдиний вхід, без переказу своїми словами —
патерн, еквівалентний:

```
claude -p "implement the ticket in materials/task-feature-ticket.md"
```

У цій сесії агент прочитав тікет, реалізував функцію й тести, прогнав
`npm test` + `npm run typecheck`, і виніс зміну в окрему гілку та PR.

## Що реалізовано

- `app/src/text-utils.ts`: додано `capitalizeWords(input: string): string` —
  перша літера кожного слова велика, решта маленькі. "Слово" = послідовність
  non-whitespace; трансформуються лише `\S+`-фрагменти через `String.replace`,
  тому оригінальні пробіли (множинні / початкові / кінцеві) зберігаються
  дослівно й **не схлопуються** (ключова вимога тікета).
- `app/src/text-utils.test.ts`: 6 unit-тестів — усі 5 прикладів із таблиці
  тікета + edge cases (порожній рядок, рядок із самих пробілів).
- Публічні сигнатури `slugify` / `truncate` / `parseTags` не змінені.

## Спостереження: CLI vs IDE-агент

- **Скоуп і контроль над diff:** CLI-агент природньо працює на рівні гілок/
  комітів — легко ізолював саме FEAT-102 в окрему гілку від `main` (stash →
  branch → застосувати лише потрібні зміни → commit → PR), тоді як IDE-агент
  (Cursor, Task B) працює "тут і зараз" у відкритому робочому дереві й
  ізоляцію diff доводиться робити руками поза інструментом.
- **Патерн тікет → PR "під ключ":** CLI пройшов увесь ланцюжок без відкритого
  редактора — прочитав тікет, написав код і тести, прогнав перевірки, створив
  гілку, запушив і відкрив PR через `gh` однією безперервною послідовністю.
  Це саме той headless-сценарій, де CLI сильніший (перегукується з
  висновком Task D: Claude Code — secondary-інструмент для автоматизації).
- **Strict typecheck зловив баг до PR:** перша версія (`word[0]`) впала на
  `noUncheckedIndexedAccess` (`Object is possibly 'undefined'`); агент замінив
  на `charAt(0)` до коміту. Показовий приклад, чому в pipeline варто ганяти
  `typecheck`, а не лише тести (тести були б зелені й з `word[0]`).
- **Придатність для CI:** ця послідовність (тікет-файл → агент → `npm test` +
  `typecheck` → PR) відтворювана в CI/CD без людини у циклі — на відміну від
  IDE-флоу Task B, який передбачає інтерактивну людину в редакторі.
