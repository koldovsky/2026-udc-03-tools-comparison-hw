## Спостереження: CLI vs IDE-агент

Патерн «тікет → CLI → готовий код» виявився помітно лінійнішим за роботу з IDE-агентом у Task B: не потрібно відкривати файли, приймати diff-и чи перемикатись між вікнами — агент сам прочитав тікет, знайшов потрібний файл, написав код і тести, перевірив `npm test`. Єдина відмінність від IDE-підходу — відсутність візуального diff перед застосуванням змін. Для повторюваних, добре описаних задач (як цей тікет) CLI-агент виграє за швидкістю.Для задач, де важливий перегляд кожного кроку — IDE-агент зручніший.
