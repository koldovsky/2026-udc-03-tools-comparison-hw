Тікет: `materials/task-feature-ticket.md`

## CLI-агент

Claude Code (той самий агент, що виконував Task B/C у цій сесії).

## Команда, яку запустили

Тікет був переданий не через окремий CLI-виклик у стилі
`claude -p "implement materials/task-feature-ticket.md"` (вправа виконувалась
всередині вже запущеної сесії агента, а не як новий процес), але вміст
тікета було прочитано та реалізовано "як є" — без переказу вимог своїми
словами, що й було метою:

```bash
# еквівалент з чистого терміналу, який відтворює цей самий флоу:
claude -p "implement the ticket in materials/task-feature-ticket.md"
```

## Що реалізовано

Додано `capitalizeWords(input: string): string` в `app/src/text-utils.ts`
одразу після `parseTags`, реалізовано через `input.replace(/\S+/g, ...)` —
це зберігає всі пробіли (включно з початковими/кінцевими та багаторазовими
внутрішніми) без змін, оскільки regex зачіпає лише non-whitespace-послідовності
("слова"), а не самі пробіли. Додано 6 unit-тестів у
`app/src/text-utils.test.ts`, що покривають усі приклади з таблиці тікета
(`"WIRELESS mouse"` → `"Wireless Mouse"`, `"red   t-shirt"` → `"Red   T-shirt"`
зі збереженням потрійного пробілу, збереження leading/trailing пробілів,
порожній рядок, рядок із самих пробілів, і ідемпотентність на вже
Title-Case вводі). Сигнатури `slugify`/`truncate`/`parseTags` не зачіпались.

## Спостереження: CLI vs IDE-агент

На відміну від Task B (де порівнювались кілька інструментів на одній і тій
самій задачі з обов'язковим скиданням стану між спробами), тут не було
конкуруючих спроб — один прохід одним CLI-агентом. Головна відмінність
патерну "тікет → код → PR" від IDE-агента — тікет можна віддати без жодного
редагування чи переказу ("як є"), і весь цикл (читання тікета → реалізація →
тести → готовність до PR) відбувається без відкриття файлів вручну в
редакторі; це особливо природно лягає на CI-подібний, повторюваний
воркфлоу (напр. "агент, реалізуй цей тікет і відкрий PR" як крок у
пайплайні), тоді як IDE-агент (Task B) більше орієнтований на інтерактивне
рев'ю кожної зміни в диффі до застосування.
